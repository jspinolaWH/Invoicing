## Where to Find This Feature

This feature spans three areas of the app. The reporting data fields themselves are visible on the Billing Event detail page under the Details tab. Structured export to external tools is available from the Billing Events list via the Export button. Field configuration (custom labels and on/off toggles) lives in the Customers & Config section under Reporting Field Configuration.

## User Flows by Acceptance Criterion

---

### AC1 — Data fields are linked to events at entry time and remain traceable

1. Click **Operations** in the top navigation bar.
2. In the left sidebar, click **Billing Events**.
3. You will see a table of all billing events. Find any event and click the **View** button on its row.
4. On the event detail page, click the **Details** tab (it is selected by default).
5. Scroll down through the details grid. You will see fields including **Accounting Account**, **Cost Center**, **Waste Type**, **Location**, **Municipality**, **Receiving Site**, **Product Group**, **Responsibility Area**, and **Service Responsibility** — all shown exactly as they were recorded at the time the event was created or received from the source system.
6. You will also see a **Created** timestamp and a **Last Modified** timestamp, confirming when the data was linked and who set it.

**Confirms requirement:** Every reporting field is displayed on the event detail page with the values that were assigned at creation time and cannot be lost as the event moves through its lifecycle.

**UI Status:** ✅ VISIBLE

---

### AC2 — Automatic data collection for internal reporting

1. Click **Operations** in the top navigation bar.
2. In the left sidebar, click **Billing Events**.
3. Click the **View** button on any event row.
4. On the event detail page, click the **Details** tab.
5. In the details grid you will see **Accounting Account** (showing the code and name resolved from the product), **Cost Center** (showing the composite code resolved via allocation rules), **Resolved Cost Centre** (the final resolved code), and **Classification** (the legal classification auto-resolved from the product configuration).
6. A note on the Create Billing Event form reads "Accounting account, cost center, and legal classification are resolved automatically at save time." This confirms the system enriches events without user input.

**Confirms requirement:** The event detail page shows that accounting classification and cost allocation data were assigned automatically at the time of event creation, with no manual entry required from the user.

**UI Status:** ✅ VISIBLE

---

### AC3 — Each event is enriched at creation time with required metadata

1. Click **Operations** in the top navigation bar.
2. In the left sidebar, click **Billing Events**.
3. Click the **+ New Event** button in the top right.
4. You will see the Create Billing Event form. Under the **Customer & Contract** section, select a product and a date.
5. A preview section labelled **Auto-Resolved Values (read-only)** will appear immediately, showing the resolved VAT rate. Below it, a note confirms: "Accounting account, cost center, and legal classification are resolved automatically at save time."
6. Fill in the **Waste Classification** section with a Waste Type and Receiving Site (optional fields).
7. Click **Create Event**. You will be taken to the new event's detail page.
8. On the detail page, you will see that **Accounting Account**, **Cost Center**, and **Classification** have been populated automatically from the product and date configuration, confirming enrichment at creation time.

**Confirms requirement:** The form interface and the resulting event detail confirm that metadata is resolved and stored automatically the moment an event is created.

**UI Status:** ✅ VISIBLE

### AC4 — Comprehensive data fields visible on each event

1. Click **Operations** in the top navigation bar.
2. In the left sidebar, click **Billing Events**.
3. Click **View** on any billing event row.
4. On the event detail page, click the **Details** tab.
5. Scroll through the details grid. You will see all ten reporting fields called out in the requirement:
   - **Accounting Account** — shown as code and name (e.g. "40001 — waste collection revenue")
   - **Responsibility Area** — text value
   - **Product Group** — text value
   - **Waste Type** — text value
   - **Service Responsibility** — text value
   - **Location** — the location ID
   - **Municipality** — the municipality ID
   - **Project** — the project ID (maps to project code)
   - **Cost Center** — the composite cost center code
   - **Receiving Site** — text value

**Confirms requirement:** All ten data fields listed in the requirement are present and visible on the billing event detail page, displaying the values stored at the time of event creation.

**UI Status:** ✅ VISIBLE

---

### AC5 — Integration with external reporting tools via structured export

1. Click **Operations** in the top navigation bar.
2. In the left sidebar, click **Billing Events**.
3. Click the **Export** button in the top right of the page header.
4. An "Export Billing Events" modal will appear. Enter a **Date From** and **Date To** (both required). Optionally filter by Status, Customer Number, or Municipality ID.
5. Click **Export**.
6. The browser will download a JSON file named `billing-export-[from]-[to].json`. This file contains all reporting fields for every billing event in the selected date range, structured for compatibility with external finance and BI tools.

**Confirms requirement:** The Export button on the Billing Events page produces a structured JSON download that can be ingested by any third-party financial or reporting system, satisfying the requirement for seamless data export without manual adjustments.

**UI Status:** ✅ VISIBLE

---

### AC6 — Scalable and configurable reporting framework

1. Click **Customers & Config** in the top navigation bar.
2. Look for a **Reporting Fields** link in the left sidebar. Note: this page is accessible via the direct URL `/config/reporting-fields` — it is not currently listed in the sidebar nav, so navigate there directly if the link is not shown.
3. You will see the **Reporting Field Configuration** page with a table listing all ten reporting fields (Accounting Account, Responsibility Area, Product Group, Waste Type, Service Responsibility, Location, Municipality, Project Code, Cost Center, Receiving Site).
4. For each field, you can:
   - Type a **Custom Label** in the text input (replacing the system default label with your organisation's terminology). The change saves automatically when you click outside the field.
   - Adjust the **Display Order** number to control the sequence in which fields appear in exports.
   - Check or uncheck the **Enabled** checkbox to include or exclude that field from billing event exports.
5. To revert a field to its default settings, click the **Reset** button on that row.

**Confirms requirement:** Administrators can customise each reporting field's label and enable/disable individual fields, allowing the system to be adapted to any organisation's reporting terminology and requirements without code changes.

**UI Status:** ⚠️ PARTIAL

The Reporting Field Configuration page exists and is fully functional, but it has no sidebar navigation entry under Customers & Config. Users cannot discover it through the normal sidebar menu — they must know the direct URL. This is a navigation discoverability gap. The feature itself is complete.
