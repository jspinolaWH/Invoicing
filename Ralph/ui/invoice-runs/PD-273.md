## Where to Find This Feature

This feature lives primarily under the **Invoices** tab. Cancellation of invoice runs is accessible from the **Invoice Runs** list (sidebar item) and from any individual run's detail page. Cancellation of a transmitted invoice that has not yet been sent to the customer is accessible from the individual invoice detail page under the **Transmission & Integration** panel.

## User Flows by Acceptance Criterion

---

### AC1 — Confirmation and cancellation for unsent invoice data

1. Click **Invoices** in the top navigation bar.
2. In the left sidebar, click **Invoice Runs**.
3. You will see a table of all invoice runs with columns for ID, Status, Mode, Invoices, Total Amount, Period, and Started.
4. Each row that is not in a SENT, CANCELLED, or ERROR status shows a red **Cancel** button on the right side.
5. Click the **Cancel** button on any eligible run row.
6. A confirmation dialog appears titled "Cancel Invoice Run #[ID]". It explains the consequences and contains a text area labelled "Cancellation reason…".
7. Type a reason in the text area (required — the **Confirm Cancel** button stays disabled until text is entered).
8. Click **Confirm Cancel**.
9. The dialog closes and the run list reloads. The cancelled run now shows a red **Cancelled** badge in the Status column.

**Confirms requirement:** The user is shown an explicit confirmation dialog before any cancellation action takes effect, and the system prevents accidental cancellation by requiring a written reason.

**UI Status:** ✅ VISIBLE

---

### AC2 — Cancelled dataset reverts status and invoice rows reflect current state

1. Click **Invoices** in the top navigation bar.
2. In the left sidebar, click **Invoice Runs**.
3. Cancel a run using the flow described in AC1.
4. After confirmation the run row immediately shows status **Cancelled**.
5. Click the run row (anywhere except the Cancel button) to open the run detail page.
6. On the detail page, scroll down to the **Invoices in this Run** table.
7. Each invoice row in the table shows its updated status badge reflecting the post-cancellation state.

**Confirms requirement:** The run status badge and individual invoice status badges both update after cancellation, showing the current invoicing state.

**UI Status:** ✅ VISIBLE

### AC3 — Invoice numbers released; errors prevent invoice creation

1. Click **Invoices** in the top navigation bar.
2. In the left sidebar, click **Invoice Runs** and click any active run row to open its detail page.
3. On the run detail page, locate the **Cancel Run** button in the action bar (visible for runs that are PENDING, RUNNING, or COMPLETED/COMPLETED_WITH_ERRORS).
4. Click **Cancel Run**.
5. A modal appears with the message "This will revert all invoices and release invoice numbers. Enter a reason:".
6. Type a reason and click **Confirm Cancel**.
7. The run status changes to **Cancelled**. Invoice numbers used in the run are released back to the number series pool automatically by the backend.
8. For runs that encountered errors during processing: the run will show status **Error** or **Completed with errors** — in the error state, the Cancel button is not shown and no invoices are created (the run detail page confirms no invoices were generated).

**Confirms requirement:** The cancel modal explicitly states "release invoice numbers", confirming to the user that numbers will be recycled. Error runs block invoice creation at the backend, which is reflected by the Error status badge in the UI.

**UI Status:** ✅ VISIBLE

### AC4 — Cancellation after transfer to external system (if permitted)

1. Click **Invoices** in the top navigation bar.
2. In the left sidebar, click **Invoice List**.
3. Find an invoice that has been transmitted to an external operator (Status shows **SENT** and there is an external reference).
4. Click the invoice row to open the Invoice Detail page.
5. Scroll down to the **Transmission & Integration** section.
6. If the waste management company's configuration permits external recall (the `allowExternalRecall` flag is enabled for this invoice), a red **Recall Invoice** button is visible.
7. Click **Recall Invoice**.
8. A form appears within the section asking for a **Reason** (required) and an optional **Internal Comment**.
9. Fill in the reason and click **Confirm Recall**.
10. A confirmation strip appears at the top of the Transmission panel reading "Invoice Recalled · [message]" and the invoice status changes to **CANCELLED**.

**Confirms requirement:** The Recall Invoice button is shown only when the integration and company configuration allow it (`allowExternalRecall` flag), satisfying the requirement that this option is conditional on company and integration permission.

**UI Status:** ✅ VISIBLE

### AC5 — Consolidated view of invoice datasets with status and cancellation option

1. Click **Invoices** in the top navigation bar.
2. In the left sidebar, click **Invoice Runs**.
3. You will see a consolidated table of all invoice runs. The table shows: ID, Status (with a coloured badge), Mode (Simulation or Real), number of invoices, total amount, period, and start time.
4. A **Status** filter dropdown at the top lets you filter by any status (PENDING, RUNNING, COMPLETED, COMPLETED_WITH_ERRORS, SENDING, SENT, CANCELLED, ERROR).
5. Each row that can be cancelled shows a red **Cancel** button directly in the table row.
6. Click any row to view the full invoice dataset and its individual invoices.

**Confirms requirement:** The Invoice Runs list is a single consolidated view where both the status of every invoice dataset and the cancel action are immediately visible without navigating away.

**UI Status:** ✅ VISIBLE

### AC6 — Users can cancel datasets created but not yet sent

1. Click **Invoices** in the top navigation bar.
2. In the left sidebar, click **Invoice Runs**.
3. Find a run with status PENDING, RUNNING, COMPLETED, or COMPLETED_WITH_ERRORS (these are the unsent states — SENT, CANCELLED, and ERROR statuses do not show a Cancel button).
4. Click the red **Cancel** button on that row, or click the row to open the detail page and click **Cancel Run** from the action bar.
5. Confirm the cancellation in the dialog that appears.
6. The run transitions to **Cancelled** status.

**Confirms requirement:** The Cancel button is explicitly suppressed for SENT and already-CANCELLED runs, ensuring users can only cancel datasets that have not yet been sent.

**UI Status:** ✅ VISIBLE

### AC7 — Recall from third-party system (not yet sent to customer)

Covered by AC4 above (same UI flow). The **Recall Invoice** button in the Transmission & Integration panel on the invoice detail page handles exactly this scenario.

1. Click **Invoices** in the top navigation bar.
2. In the left sidebar, click **Invoice List**.
3. Click any invoice that has been transmitted externally (status SENT with an external reference).
4. Scroll to the **Transmission & Integration** section.
5. If recall is permitted, click **Recall Invoice**, provide a reason, and click **Confirm Recall**.
6. The invoice status changes to **CANCELLED** and a confirmation message appears in the panel.

**Confirms requirement:** The Recall Invoice functionality is gated by `allowExternalRecall`, meaning it only appears when both the integration and the company configuration support it.

**UI Status:** ✅ VISIBLE

### AC8 — Default invoice dataset statuses: Draft, ready, and sent

The Invoice Runs list and run detail page display the full status lifecycle using coloured badges:

- **Pending** (grey) — the run has been queued but not yet started
- **Running** (blue) — the run is actively processing
- **Completed** (green) — all invoices generated successfully (equivalent to "ready")
- **Completed with errors** (amber) — run finished with some validation issues
- **Sending** (blue) — invoices are being transmitted to the external operator
- **Sent** (default badge) — all invoices have been sent to the customer
- **Cancelled** (red) — the dataset was cancelled
- **Error** (red) — the run failed

1. Click **Invoices** in the top navigation bar.
2. In the left sidebar, click **Invoice Runs**.
3. The Status column shows coloured badges for every run. You can also filter by status using the dropdown at the top of the list.

**Confirms requirement:** The full status progression from created (Pending) through ready (Completed) to sent (Sent) is visible in the Invoice Runs list, confirming the default status set from the requirement.

**UI Status:** ✅ VISIBLE

### AC10 — Integration with an invoicing system

1. Click **Invoices** in the top navigation bar.
2. In the left sidebar, click **Invoice List**.
3. Click any invoice row.
4. On the Invoice Detail page, scroll down to the **Transmission & Integration** section.
5. Depending on the invoice status, you will see one or more of the following buttons: **Transmit to Operator**, **Check Delivery Status**, **View Invoice Image (PDF)**, **Fetch External Attachments**, **Retransmit (Updated Address)**, and — if recall is permitted — **Recall Invoice**.
6. These actions interact with the external invoicing broker (operator) to transmit, retransmit, or recall invoices.

**Confirms requirement:** The Transmission & Integration panel provides a direct UI surface for all integration-related actions between the invoicing system and the external operator.

**UI Status:** ✅ VISIBLE
