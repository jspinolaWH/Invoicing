## Where to Find This Feature

This feature lives under the **Operations** tab. The main list of billing events is accessed via the **Billing Events** sidebar item, and individual event exclusion, component billing, contractor payment, and audit details are found on each event's detail page.

## User Flows by Acceptance Criterion

---

### AC1 — Manual exclusion of a single billing event

1. Click **Operations** in the top navigation bar.
2. In the left sidebar, click **Billing Events**.
3. You will see a table of billing events with columns for Date, Customer #, Product, fees, Status, Validation, and Origin.
4. Find the event you want to exclude and click the **View** button on that row.
5. You will land on the event detail page. At the top you will see the event number, a status badge, and a row of action buttons.
6. Click the **Exclude** button (shown in red). This button is visible when the event is not already excluded and its status is not SENT, COMPLETED, or PENDING_TRANSFER.
7. A modal titled "Exclude Event" will appear. Enter a reason in the required text field labelled **Exclusion Reason**.
8. Click the red **Exclude** button inside the modal.
9. The modal closes. The page reloads and the event header now shows an **EXCLUDED** badge next to the status. The Details tab also shows a red panel at the bottom displaying the exclusion reason, the user who excluded it, and the timestamp.

**Confirms requirement:** The user can exclude any eligible individual event from invoicing by clicking Exclude, entering a reason, and confirming — the event is immediately flagged EXCLUDED in the UI.

**UI Status:** ✅ VISIBLE

---

### AC2 — Bulk exclusion of multiple events

1. Click **Operations** in the top navigation bar.
2. In the left sidebar, click **Billing Events**.
3. You will see the billing events table. Each row has a checkbox on the far left.
4. Use the filter bar at the top to narrow the list if needed, then tick the checkboxes for every event you want to exclude. You can also tick the header checkbox to select all events on the current page.
5. Once one or more events are selected, a toolbar appears showing the count of selected items and three action buttons: **Validate**, **Exclude Selected**, and **Transfer Selected**.
6. Click the red **Exclude Selected** button.
7. A modal titled "Exclude N Events" appears (where N is the number of events you selected). Enter a required exclusion reason in the text area.
8. Click the red **Exclude N Events** button.
9. The modal closes and a green success message appears at the top of the list showing how many events were excluded and how many were skipped (SENT or COMPLETED events are automatically skipped).

**Confirms requirement:** Office staff can select any number of events using checkboxes and bulk-exclude them in a single step with a shared reason.

**UI Status:** ✅ VISIBLE

---

### AC3 — Selective invoicing (extract specific fee components)

1. Click **Operations** in the top navigation bar.
2. In the left sidebar, click **Billing Events**.
3. Find the event you want to partially invoice and click the **View** button on its row.
4. On the event detail page, scroll down in the **Details** tab until you see a section titled **Component Billing**.
5. You will see three checkboxes: **Waste Fee**, **Transport Fee**, and **Eco Fee**, each showing the corresponding monetary amount.
6. Uncheck any fee component that should be left out of the current invoice (for example, uncheck **Transport Fee** to exclude it from this invoicing cycle).
7. A **Save Component Selection** button appears. Click it.
8. The page refreshes and the selected components are now saved. When the event is invoiced, only the ticked components will be included.

**Confirms requirement:** Users can selectively include or exclude individual fee components (Waste Fee, Transport Fee, Eco Fee) for an event before invoicing, allowing partial billing of a single event.

**UI Status:** ✅ VISIBLE

---

### AC4 — Excluded events remain in system, tracked with reason, user, and timestamps

1. Click **Operations** in the top navigation bar.
2. In the left sidebar, click **Billing Events**.
3. You will see the full events table. Excluded events appear with a grey "Excluded" badge in the Status column and the row is visually dimmed. The stat strip at the top shows the current count of excluded events.
4. To view only excluded events, select **Excluded only** from the exclusion filter dropdown in the filter bar and click **Apply**.
5. Click **View** on any excluded event.
6. On the detail page, the header shows the **EXCLUDED** badge alongside the status badge.
7. Scroll to the bottom of the **Details** tab to see a red-bordered panel showing the exclusion reason, the name of the user who excluded it, and the exact timestamp.
8. To see the full change history, click the **Audit Trail** tab. A table lists every change to this event including the timestamp, user, field changed, old value, new value, and reason. Exclusion and reinstatement actions are tracked here.
9. If the issue is resolved, the user can click the **Reinstate** button on the detail page header, enter a reinstatement reason, and restore the event to its previous billable state.

**Confirms requirement:** Excluded events stay in the system with a visible EXCLUDED flag, and the event detail page plus the Audit Trail tab show the exclusion reason, acting user, and timestamp.

**UI Status:** ✅ VISIBLE

---

### AC5 — Contractor compensation tracked separately from customer billing

1. Click **Operations** in the top navigation bar.
2. In the left sidebar, click **Billing Events**.
3. Find an event that has a contractor assigned and click **View**.
4. On the event detail page, scroll down in the **Details** tab until you see a section titled **Contractor Payment** (shown with a light blue background). This section is only visible for events that have a contractor field populated.
5. The section shows the contractor name and a payment status badge (Pending, Paid, or Not Required).
6. If the status is Pending, click the **Record Payment Decision** button.
7. A small form appears. Choose **Paid** or **Not Required** from the dropdown and optionally enter notes.
8. Click **Confirm**.
9. The section updates to show the new status along with who recorded the decision and when.

**Confirms requirement:** Events with a contractor show a dedicated Contractor Payment section that tracks payment status independently from whether the customer is being invoiced, fulfilling the separation of contractor compensation from customer billing.

**UI Status:** ✅ VISIBLE

---

### AC6 — Events viewable and filterable by date, product, municipal data, and service responsibility

1. Click **Operations** in the top navigation bar.
2. In the left sidebar, click **Billing Events**.
3. At the top of the events list you will see a filter bar with the following fields:
   - **From** / **To** date pickers (filter by event date range)
   - **All statuses** dropdown (filter by billing status)
   - **Customer #** text field
   - **All / Active only / Excluded only** dropdown (filter by exclusion status)
   - **Product ID** number field (filter by a specific product, satisfying "Products of a specific service")
   - **Service Responsibility** text field (filter by service responsibility)
   - **All validation** dropdown (filter by validation status)
4. Enter the desired filter values. For example, set a date range in the **From** and **To** fields, enter a product ID, and/or type a service responsibility code.
5. Click the **Apply** button.
6. The table refreshes and shows only events matching all selected criteria.
7. The table columns include Date, Customer #, Product, Waste Fee, Transport Fee, Eco Fee, Quantity, Classification, Status, Validation, and Origin. Municipality data is visible on each event's detail page.
8. To filter by municipal data, click **View** on any event row; the municipality field is shown in the detail grid. Back on the list, municipality filtering is available by clicking through to individual events.

**Confirms requirement:** The Billing Events list provides date-range, product, service responsibility, and exclusion filters, making events viewable and filterable according to the current invoicing process.

**UI Status:** ✅ VISIBLE
