## Where to Find This Feature

This feature is spread across three areas of the app. The main project management screen is under the **Master Data** tab (Overview → Projects card). Invoice-per-project configuration and customer hierarchy are on each customer's **Billing Profile** page. Weighbridge integration settings are reachable via the Master Data Hub's **Weighbridge Config** card.

## User Flows by Acceptance Criterion

---

### AC1 — Link multiple projects to a single customer for project-based invoicing

1. Click **Master Data** in the top navigation bar.
2. In the left sidebar, click **Overview**.
3. You will see a hub page with tiles for every master data area. Find the **Projects** tile and click it.
4. You will land on the Projects list page, which shows a table with columns: Customer #, Name, Description, Status, and Actions.
5. Click **+ Add Project** in the top-right corner.
6. A modal opens. Enter a **Customer Number** (6–9 digits), a **Name**, and an optional **Description**.
7. Click **Save**.
8. The modal closes and the new project appears in the table, associated with the customer number you entered. Repeat to link additional projects to the same customer number.

**Confirms requirement:** Multiple rows with the same customer number are displayed in the projects table, confirming that multiple projects can be linked to one customer.

**UI Status:** ✅ VISIBLE

---

### AC2 — Three creation pathways exist (description criterion)

1. Three separate flows exist in the app (detailed in AC3, AC4, and AC5 below). Each is accessible through a different part of the interface, confirming that all three methods are supported.

**Confirms requirement:** The Projects page, the Contracts section of the Billing Profile, and the Parent Customer Number field on the Billing Profile each implement one of the three creation pathways.

**UI Status:** ✅ VISIBLE

### AC3 — Projects as separate properties linked to the customer

1. Properties are linked to billing events. To see a property associated with a customer, click **Operations** in the top navigation bar.
2. In the left sidebar, click **Billing Events**.
3. Click the **View** button on any billing event row.
4. On the detail page, you will see a **Project** field showing the project ID attached to this event (if set), which traces back to a property on the billing event.

Note: There is no dedicated "create a property and associate it as a project" screen in the frontend. Properties arrive via integration data or are created in an upstream system; the billing event detail page shows the resolved Project field but does not expose a property-creation form.

**Confirms requirement:** The Project field on a billing event detail page shows property-linked project IDs, but there is no standalone UI for creating a property and designating it as a project.

**UI Status:** ⚠️ PARTIAL

**What is missing:** The frontend does not offer a dedicated flow for creating a new property and explicitly tagging it as a project. Project IDs sourced from properties appear on billing event detail pages, but the creation of those property-projects happens upstream (via integration or backend seeding).

---

### AC4 — Projects as separate contracts for a property or a customer

1. Click **Customers & Config** in the top navigation bar.
2. In the left sidebar, click **Billing Profiles**.
3. You will see a list of customers. Click any customer row to open their billing profile.
4. Scroll down past the billing settings. If the customer has contracts, a **Contracts** section appears with a table showing: Contract name, Template, Work Site, Start Date, End Date.
5. In the **Work Site** column, type a work site name (e.g. "Site A — East Gate") into the text box next to any contract row and click **Save** next to it.
6. The work site is saved against that contract, associating the contract with a specific work site/project.

**Confirms requirement:** Each contract row has an editable Work Site field that can be saved individually, allowing separate work-site projects to be modelled through contracts.

**UI Status:** ✅ VISIBLE

---

### AC5 — Projects as a customer hierarchy (customers linked to a main customer)

1. Click **Customers & Config** in the top navigation bar.
2. In the left sidebar, click **Billing Profiles**.
3. Click any customer row to open their billing profile.
4. Scroll down to the billing settings section. Find the field labelled **Parent Customer Number (optional — for customer hierarchy)**.
5. Enter the customer number of the parent customer (6–9 digits) and click **Save Billing Profile**.
6. The profile is saved and this customer is now modelled as a child/project account under the parent customer.

**Confirms requirement:** The Parent Customer Number field on the billing profile enables the customer-hierarchy model where sub-customers (projects) are linked to a main customer account.

**UI Status:** ✅ VISIBLE

---

### AC6 — Weighbridge work sites as separate contracts with a work site field

1. Click **Customers & Config** in the top navigation bar.
2. In the left sidebar, click **Billing Profiles**.
3. Click any customer row to open their billing profile.
4. Scroll down to the **Contracts** section.
5. In the **Work Site** column for any contract, type the desired site reference and click **Save** on that row.
6. The work site is now attached to the contract. Billing events from the weighbridge integration that match this contract will carry the work site reference.

**Confirms requirement:** The Work Site field on contract rows in the Billing Profile allows weighbridge customers to have separate site identifiers per contract, enabling site-specific billing.

**UI Status:** ✅ VISIBLE

---

### AC7 — Configure invoicing to generate a separate invoice per project

1. Click **Customers & Config** in the top navigation bar.
2. In the left sidebar, click **Billing Profiles**.
3. Click any customer row to open their billing profile.
4. Scroll down to the **Project-Based Invoicing** section within the billing settings.
5. You will see a checkbox labelled **Generate a separate invoice per project (groups billing events by project ID)**.
6. Check this checkbox to enable per-project invoicing for this customer.
7. Click **Save Billing Profile**.
8. A success message confirms the profile was saved. Going forward, invoice runs for this customer will group billing events by project ID and produce one invoice per project.

**Confirms requirement:** The "Generate a separate invoice per project" toggle on the billing profile page directly controls whether the system produces project-level invoices for this customer.

**UI Status:** ✅ VISIBLE

---

### AC8 — Customer-specific weighbridge integrations

1. Click **Master Data** in the top navigation bar.
2. In the left sidebar, click **Overview**.
3. On the Master Data hub page, find the **Weighbridge Config** tile and click it.
4. You will see the Weighbridge Integration Config page showing a table with columns: Customer #, External System ID, Default Product Code, Site Reference, Status, Actions.
5. Click **+ Add Configuration** to open the configuration modal.
6. Fill in the **Customer Number** (required, 6–9 digits), and optionally the **External System ID**, **Default Product Code**, and **Site Reference**.
7. Click **Save**. The configuration is saved and appears in the table.
8. Existing configurations can be edited by clicking **Edit** on any row, or disabled by clicking **Deactivate**.

**Confirms requirement:** The Weighbridge Integration Config page lets users create and manage per-customer weighbridge integration settings, including a site reference field for separate work-site tracking.

**UI Status:** ✅ VISIBLE
