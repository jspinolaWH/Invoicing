## Where to Find This Feature

This feature is spread across the **Master Data** tab and the **Operations** tab. The master data configuration pages (Cost Centers, Accounting Accounts, VAT Rates, Products, Price Lists) let office users define and maintain the financial framework, while the Billing Event detail page shows exactly which account and cost center were resolved for any individual event, and the Accounting Report page provides the aggregated view for reporting.

## User Flows by Acceptance Criterion

---

### AC1 — View background financial data on a billing event

1. Click **Operations** in the top navigation bar.
2. In the left sidebar, click **Billing Events**.
3. You will see a table of all billing events with columns for Date, Customer, Product, Status, and more.
4. Click the **View** button on any billing event row.
5. You are now on the Billing Event Detail page. The **Details** tab is active by default.
6. Scroll down through the detail grid. You will see labeled fields for:
   - **Accounting Account** — showing the account code and name linked to this event (e.g. "30002 — Waste treatment revenue")
   - **Cost Center** — showing the composite cost center code linked to this event (e.g. "WASTE-HEL-MUNI")
   - **Resolved Cost Centre** — the final resolved cost center code after any allocation rules are applied
   - **Project** — any project linked to this event
7. Scroll further to the **VAT Details** section (visible when VAT data is present). You will see:
   - **VAT Rate Code** — the code of the applicable VAT rate
   - **Effective Rate** — the percentage (e.g. 25.5%)
   - **Net Amount**, **VAT Amount**, and **Gross Amount**
8. The price component breakdown (Waste Fee, Transport Fee, Eco Fee) is also visible in the main grid.

**Confirms requirement:** The user can see the exact accounting account, cost center, resolved cost center, VAT rate, and monetary amounts attached to any billing event, confirming that background financial data is captured and accessible per event.

**UI Status:** ✅ VISIBLE

### AC2 — Manage cost centers, VAT rates, accounting accounts, and product financial settings

#### Managing Cost Centers

1. Click **Master Data** in the top navigation bar.
2. In the left sidebar, click **Cost Centers**.
3. You will see a table listing all cost centers, with columns for Composite Code, Product segment, Reception segment, Responsibility segment, and Description.
4. To add a new cost center, click **+ Add Cost Center** in the top-right corner.
5. A modal dialog appears with three required fields: **Product Segment**, **Reception Segment**, and **Responsibility Segment**, plus an optional **Description**. As you type, the composite code preview updates automatically.
6. Click **Save**. The new cost center appears in the table.
7. To modify an existing cost center, click **Edit** on any row. The same modal opens pre-filled. Edit as needed and click **Save**.
8. To remove a cost center, click **Delete** on any row. A confirmation dialog appears; confirm to delete.

#### Managing Accounting Accounts

1. Click **Master Data** in the top navigation bar.
2. In the left sidebar, click **Accounting**.
3. You will see a table of accounting accounts with columns for Code, Name, Valid From, Valid To, and Created By.
4. Optionally, use the **Filter by date** field to show only accounts active on a specific date.
5. Click **+ Add Account** to add a new account. The modal requires a **Code**, a **Name**, and a **Valid From** date. **Valid To** is optional (leave blank for an open-ended account).
6. Click **Save**. The new account appears in the table.
7. Click **Edit** on an existing row to update it. Click **Delete** to remove it after confirming the prompt.

#### Managing VAT Rates

1. Click **Master Data** in the top navigation bar.
2. In the left sidebar, click **VAT Rates**.
3. You will see a table of VAT rates with columns for Code, Rate (%), Valid From, Valid To, and Created By.
4. Optionally use the **Filter by event date** field to see which rates would apply on a given date.
5. Click **+ Add VAT Rate** to add a rate. The modal requires a **Code**, a **Rate (%)**, and a **Valid From** date.
6. Click **Save**. Click **Edit** or **Delete** on any row to modify or remove a rate.

#### Assigning Accounts and Cost Centers to Products

1. Click **Master Data** in the top navigation bar.
2. In the left sidebar, click **Products**.
3. You will see a table of products with columns for Code, Pricing Unit, Reverse Charge VAT, **Default Account**, **Default Cost Center**, Price List, and Translations.
4. Click **Edit** on any product row (or **+ Add Product** to create a new one).
5. In the modal, find the **Default Accounting Account** field — type part of an account code or name to search and select one from the autocomplete dropdown.
6. Find the **Default Cost Center** field — use the dropdown to select a cost center composite code.
7. Optionally assign a **Price List** in the same modal.
8. Click **Save**. The table now shows the assigned Default Account and Default Cost Center codes next to the product.

**Confirms requirement:** Office users have full create/read/update/delete capability for cost centers, accounting accounts, VAT rates, and product-level financial settings in one place.

**UI Status:** ✅ VISIBLE

### AC3 — View aggregated accounting and cost center report for BI/ERP integration

1. Click **Master Data** in the top navigation bar.
2. In the left sidebar, click **Overview**.
3. You will see a grid of tiles on the Master Data hub page.
4. Click the **Accounting Report** tile.
5. You are now on the Accounting Report page.
6. Enter a **From** date and a **To** date in the date fields to define the reporting period (both are optional — leave blank to report all time).
7. Click **Generate Report**.
8. The page shows two side-by-side tables:
   - **By Accounting Account** — with columns Account Code, Account Name, Net Amount (€), and number of invoice lines
   - **By Cost Center** — with columns Cost Center composite code, Description, Net Amount (€), and number of invoice lines
9. Each table shows a **Total** row summing all amounts.

**Confirms requirement:** The user can produce a breakdown of revenue and costs by accounting account and cost center for any date range, ready to be read by an external ERP or BI system.

**UI Status:** ✅ VISIBLE

### AC4 — Confirm financial data flows from event creation through reporting

1. Click **Operations** in the top navigation bar.
2. In the left sidebar, click **Billing Events**.
3. Click **+ New Event** to create a billing event (or **View** on an existing event).
4. On the Create Billing Event page, select a **Product**. Because the product has a Default Accounting Account and Default Cost Center already configured (see AC2 above), these values are automatically resolved on the event.
5. After saving and opening the event's detail page, the **Accounting Account** and **Cost Center** fields in the detail grid are already populated from the product definition — no manual entry needed.
6. Once the event progresses to invoicing, go to **Master Data > Overview > Accounting Report**, generate a report covering the event date, and confirm the event's amounts appear under the correct account and cost center rows.

**Confirms requirement:** Financial data flows automatically from event creation (product defaults) through to the accounting report without manual re-entry, confirming end-to-end data consistency.

**UI Status:** ✅ VISIBLE

---

### AC5 — Manage a large product catalog with varied pricing and accounting

1. Click **Master Data** in the top navigation bar.
2. In the left sidebar, click **Products**.
3. You will see the full product catalog in a paginated/scrollable table.
4. Each product row shows its unique **Code**, **Pricing Unit** (PCS, KG, TON, M3, LITER, METER, HOUR), **Reverse Charge VAT** flag, **Default Account**, **Default Cost Center**, and **Price List** — all independently configurable per product.
5. Click the translations count link (e.g. "2 translations ▼") on any row to expand the translations panel and manage Finnish, Swedish, and English product names.
6. Click **+ Add Product** to add a new product with its own pricing unit, account, cost center, and price list. There is no artificial limit on the number of products.
7. Click **Edit** on any row to change a product's financial settings when tariffs or regulations change.

**Confirms requirement:** The system supports an unlimited product catalog where each product independently carries its own pricing unit, accounting account, cost center, price list, and VAT classification, satisfying the requirement for large catalogs with unique financial settings per product.

**UI Status:** ✅ VISIBLE

## Orphaned UI Noticed

- **Price Lists page** (`/master-data/price-lists`) — Accessible from the Master Data hub. Price lists can be created, edited, and deleted with code, name, tariff variant, validity period, and active flag. While this page is tagged with PD-296 in its `RELATED_TASKS` array, the requirement text focuses mainly on cost centers, accounts, and VAT. The price list management itself may align more closely with a separate pricing/tariff requirement not fully described in PD-296's acceptance criteria.

- **Accounting Report** (`/reports/accounting`) — This page is reachable only from the Master Data hub overview tile, not from any sidebar item. It has no direct sidebar link, making it potentially hard for users to discover unless they know to look on the hub page.
