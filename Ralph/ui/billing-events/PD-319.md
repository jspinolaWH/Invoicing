## Where to Find This Feature

This feature lives under the **Tools** tab in the top navigation bar. It is accessed via the **Price Adjustment** sidebar item, which provides a dedicated form for retroactively changing prices on unbilled billing events.

---

## User Flows by Acceptance Criterion

---

### AC1 — Users can make retroactive price changes

1. Click **Tools** in the top navigation bar.
2. In the left sidebar, click **Price Adjustment**.
3. You will see the **Retroactive Price Adjustment** form. The page contains fields for selecting events, choosing the type of price change, and entering the new prices.
4. Fill in the **Customer Number** field with the customer whose events you want to adjust.
5. Choose your selection mode (Date Range or Specific Event IDs), fill in the required fields, choose the adjustment type, fill in the price values, then click **Preview Changes**.
6. After reviewing the preview table, click **Apply Adjustment**, then confirm in the confirmation dialog.

**Confirms requirement:** The user can open the Price Adjustment page, specify events, enter new prices, preview, and apply the changes — demonstrating that retroactive price changes are supported.

**UI Status:** ✅ VISIBLE

---

### AC2 — Selecting a date range to identify unbilled events for a time period

1. Click **Tools** in the top navigation bar.
2. In the left sidebar, click **Price Adjustment**.
3. You will see two **Selection mode** toggle buttons at the top of the form: **Date Range** and **Specific Event IDs**. By default, **Date Range** is selected.
4. In **Date Range** mode, fill in the **Event Date From** and **Event Date To** date fields to specify the time period.
5. Optionally narrow the scope by entering a **Product ID** or **Service Responsibility** value.
6. Click **Preview Changes** — the system will query all unbilled events within that date range for the given customer.
7. The preview table that appears lists all matching events, showing their current status (e.g. IN_PROGRESS vs SENT/COMPLETED), so you can see which are unbilled.

**Confirms requirement:** The date range fields and optional product/service filters let a user specify the exact time period, and the system returns all matching unbilled events for review.

**UI Status:** ✅ VISIBLE

---

### AC3 — Applying new prices by percentage or fixed euro adjustments

1. Click **Tools** in the top navigation bar.
2. In the left sidebar, click **Price Adjustment**.
3. After filling in the customer and event selection fields, look for the **Adjustment type** toggle: **Fixed Amount** or **Percentage**.
4. If you select **Fixed Amount**, three fields appear: **New Waste Fee Price**, **New Transport Fee Price**, and **New Eco Fee Price**. Enter the new euro amount for each fee component. Fields left blank retain their current price.
5. If you select **Percentage**, a single **Percentage Adjustment (%)** field appears. Enter a positive number (e.g. 10 for +10%) or a negative number (e.g. -5 for −5%).
6. Both adjustment types are available in a single toggle, making it easy to switch between them.

**Confirms requirement:** The two toggle options — Fixed Amount and Percentage — allow users to apply price changes in either format, covering both adjustment methods described in the requirement.

**UI Status:** ✅ VISIBLE

---

### AC4 — Mass updates or single event updates

1. Click **Tools** in the top navigation bar.
2. In the left sidebar, click **Price Adjustment**.
3. Notice the **Selection mode** toggle at the top: **Date Range** and **Specific Event IDs**.
4. For a **single event update**: select **Specific Event IDs**, then enter a single event ID in the text field (e.g. `101`).
5. For a **mass (batch) update**: either enter multiple comma-separated event IDs (e.g. `101, 102, 103`) in Specific Event IDs mode, or use **Date Range** mode to target an entire period — which can cover hundreds of events at once.
6. After previewing, the preview summary shows **Total events** so you can confirm how many will be affected before applying.

**Confirms requirement:** Both single-event and bulk update paths are supported via the two selection modes — specific IDs for targeted updates, and date range for broad batch changes.

**UI Status:** ✅ VISIBLE

### AC5 — System protects already-invoiced events from retroactive changes

1. Click **Tools** in the top navigation bar.
2. In the left sidebar, click **Price Adjustment**.
3. Fill in the selection and adjustment fields, then click **Preview Changes**.
4. The preview summary panel shows four statistics: **Total events**, **In-progress**, **Already billed**, and **Net delta**.
5. If any events in the selected range are already billed, a yellow warning banner appears: "Note: X already-billed event(s) will be excluded and correction copies created for re-invoicing."
6. The preview table shows each event's **Status** badge (IN_PROGRESS, SENT, COMPLETED, ERROR), making it visible which events are billable vs already invoiced.
7. When you apply, in-progress events are updated directly while already-billed events receive correction copies rather than in-place changes.

**Confirms requirement:** The preview explicitly separates in-progress (unbilled) events from already-billed events, and the apply response reports the count of each — proving that already-invoiced events are protected from direct modification.

**UI Status:** ✅ VISIBLE

### AC6 — Price adjustments are logged with reason, user, and affected events

1. Click **Tools** in the top navigation bar.
2. In the left sidebar, click **Price Adjustment**.
3. On the form, locate the **Reason** field (e.g. "Contract renegotiation") and the **Performed By** field (e.g. "john.doe") — both are part of the main form.
4. The **Internal Comment** field is required when applying changes and provides an additional audit note.
5. When **Apply Adjustment** is confirmed, the system records the adjustment run including the reason, the user who performed it, and the list of affected event IDs.
6. After a successful apply, the result panel shows the affected event IDs (if correction copies were created) along with summary counts.

**Confirms requirement:** The form requires a reason and performer identity, and the apply response surfaces the affected event IDs — providing the audit trail described in the requirement.

**UI Status:** ✅ VISIBLE

---

### AC7 — Preview tool lets users validate impact before finalising

1. Click **Tools** in the top navigation bar.
2. In the left sidebar, click **Price Adjustment**.
3. Fill in the customer, date range or event IDs, adjustment type, and price values.
4. Click **Preview Changes** (the blue button at the bottom of the form). This does not apply anything — it only fetches the projected impact.
5. A summary panel appears showing: **Total events**, **In-progress**, **Already billed**, and **Net delta** (shown in green if positive, red if negative).
6. Below the summary, a table lists every affected event with columns: **Event ID**, **Date**, **Product**, **Quantity**, **Status**, **Current Net**, **Projected Net**, and **Delta** — so you can review exactly what will change.
7. Only after reviewing the preview do you click the red **Apply Adjustment** button, which then shows a second confirmation dialog ("Are you sure? This will modify X billing event(s). This action cannot be undone.") before any changes are made.

**Confirms requirement:** The two-step workflow — Preview first, then Apply with a confirmation dialog — gives users a full impact view before anything is changed, satisfying the validation and previewing requirement.

**UI Status:** ✅ VISIBLE
