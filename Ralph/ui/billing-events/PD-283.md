## Where to Find This Feature

This feature lives under the **Operations** tab. The main entry point is the **Billing Events** list page, from which users click **+ New Event** to open the manual creation form. Once created, the event appears in the same list alongside all automated events.

## User Flows by Acceptance Criterion

---

### AC1 — Manual entry of invoicing events with auto-resolved financial metadata

1. Click **Operations** in the top navigation bar.
2. In the left sidebar, click **Billing Events**.
3. You will see a table of all billing events with a status dashboard at the top. In the top-right corner, click the **+ New Event** button.
4. The "Create Billing Event" form opens. Fill in the **Date** field (required).
5. In the **Customer & Contract** section, type a customer number or name into the **Customer** search box and select the matching customer from the dropdown.
6. Once a customer is selected, the **Contract** dropdown becomes available. Choose the appropriate contract.
7. Once a contract is selected, the **Product** dropdown populates with products available under that contract. Select the desired product (for example, "Land rent" or "Expert work").
8. As soon as both a product and a date are selected, a read-only **Auto-Resolved Values** section appears below, showing the **VAT Rate** that will be applied. A note confirms that accounting account, cost center, and legal classification are resolved automatically at save time.
9. Enter **Waste Fee**, **Transport Fee**, and **Eco Fee** in the **Pricing** section, along with **Quantity** and **Weight**.
10. Fill in any optional fields (Location, Municipality, Vehicle, Driver, Comments, Waste Type, Receiving Site, Contractor, Direction, Property Group).
11. Click **Create Event** to submit the form.
12. You are redirected to the new billing event's detail page, where all resolved financial data (accounting account, cost center, VAT rate, classification) is displayed.

**Confirms requirement:** The user can see the VAT rate auto-resolved before saving, and after saving the detail page shows the full financial classification, accounting account, cost center, and VAT details — confirming AC1.

**UI Status:** ✅ VISIBLE

### AC2 — Full integration into WasteHero for non-core service invoicing

1. Click **Operations** in the top navigation bar.
2. In the left sidebar, click **Billing Events**.
3. The Billing Events list shows events from all origins in the same table. The **Origin** column on each row indicates whether an event came from a manual entry or an automated integration.
4. Manually created events appear in the same workflow as automated events — they pass through the same statuses (Draft, In Progress, Sent, Completed, etc.) and go through the same invoicing run process.
5. Click **+ New Event** and create an event for a non-core service (for example, consulting or equipment rental) by selecting the relevant product from the product catalogue. Submit the form.
6. The new event appears in the Billing Events list with origin **MANUAL**, alongside all automated events, and flows through the same status lifecycle.

**Confirms requirement:** Manual events are indistinguishable from automated ones in the list view except for the Origin label, demonstrating full integration into the standard invoicing workflow.

**UI Status:** ✅ VISIBLE

---

### AC3 — Manual entry with product selection from the catalogue

1. Click **Operations** in the top navigation bar.
2. In the left sidebar, click **Billing Events**.
3. Click the **+ New Event** button at the top right.
4. In the **Customer & Contract** section, select a customer and then a contract.
5. The **Product** dropdown is populated with all products configured under that contract. Select the appropriate product from the list.
6. The product name is shown in the dropdown (in Finnish, English, or its code as configured). Default fees for Waste Fee, Transport Fee, and Eco Fee are pre-filled based on the selected product's defaults.
7. Click **Create Event** to save. The event is created and linked to the chosen product.

**Confirms requirement:** The product selection is a required field on the form and drives downstream financial assignments, confirming that manual events are always linked to catalogue products.

**UI Status:** ✅ VISIBLE

---

### AC4 — Product-based invoicing with automatic financial allocations

1. Follow steps 1–7 from AC3.
2. After selecting a product and a date, observe the **Auto-Resolved Values** section that appears on the form. It shows the effective **VAT Rate** determined by the product's configuration and the selected event date.
3. The form also pre-fills the **Waste Fee**, **Transport Fee**, and **Eco Fee** with the product's default prices (these can be overridden by the user).
4. Submit the form by clicking **Create Event**.
5. On the resulting detail page, scroll down to see the **VAT Details** section, which shows the VAT rate code, effective rate percentage, net amount, VAT amount, and gross amount — all computed from the product's accounting rules.
6. The **Accounting Account** and **Cost Center** fields in the detail grid show the values that were automatically assigned at save time based on the selected product.

**Confirms requirement:** The detail page explicitly shows accounting account, cost center, and VAT information resolved from the product's configuration, confirming product-based financial allocation.

**UI Status:** ✅ VISIBLE

---

### AC5 — Predefined financial allocations assigned automatically

1. Create a billing event manually as described in AC3.
2. After the event is saved and you are on the detail page, look at the main detail grid. The **Accounting Account** field shows the revenue account (code and name) that was automatically assigned. The **Cost Center** and **Resolved Cost Centre** fields show the automatically assigned cost center.
3. The **VAT Details** section below the detail grid shows the resolved VAT rate code, effective VAT rate percentage, reverse-charge flag, net amount, VAT amount, and gross amount — all assigned automatically without any user action.
4. The **Classification** field shows the legal classification (Public Law or Private) that was automatically determined.

**Confirms requirement:** Accounting account, cost center, VAT rate, and legal classification all appear in the detail view as automatically resolved values, with no manual entry of these fields needed by the user.

**UI Status:** ✅ VISIBLE

### AC6 — Support for non-core business events (land rent, expert work, consulting, etc.)

1. Click **Operations** in the top navigation bar.
2. In the left sidebar, click **Billing Events**.
3. Click **+ New Event**.
4. Select a customer and contract, then open the **Product** dropdown. Products like "Land rent", "Expert work", "Equipment rental", or any other product configured in the system catalogue appear in this list.
5. Select the desired non-core service product.
6. The form accepts any product in the catalogue regardless of whether it is related to waste collection — there is no restriction on product category in the creation form.
7. Enter the appropriate quantity, weight, and pricing details. For hourly consulting work, enter the number of billable hours in the **Quantity** field.
8. Submit with **Create Event**.

**Confirms requirement:** The product dropdown is open to any product in the catalogue, enabling invoicing of any type of service (land rent, consulting, etc.) without restriction.

**UI Status:** ✅ VISIBLE

---

### AC7 — Customisable invoice lines with flexible detail entry

1. Click **Operations** in the top navigation bar.
2. In the left sidebar, click **Billing Events**.
3. Click **+ New Event**.
4. The form is divided into multiple editable sections:
   - **Pricing**: Waste Fee, Transport Fee, and Eco Fee can each be entered independently. Default values from the product are pre-filled but the user can change any of them.
   - **Quantity & Weight**: enter the specific amount and weight for this particular event.
   - **Location**: optional Municipality, Location ID, Vehicle ID, and Driver ID.
   - **Notes**: a free-text comments field for invoice-line notes.
   - **Waste Classification**: optional Waste Type and Receiving Site.
   - **Additional Details**: Contractor, Direction, Property Group, and Shared Collection Group percentage.
5. Users can also click **Save as Template** to name and save the current form values as a reusable template for future events of the same type.
6. When hovering over **Save as Draft**, required fields shrink to only Date and Customer — the user can save a partial entry and complete it later.

**Confirms requirement:** The form provides event-specific detail fields while pre-filling pricing from the product default, offering structured flexibility consistent with the requirement.

**UI Status:** ✅ VISIBLE

---

### AC8 — Audit and reporting integration for manual events

1. Click **Operations** in the top navigation bar.
2. In the left sidebar, click **Billing Events**.
3. Find a manually created billing event in the list (identifiable by the **MANUAL** origin badge) and click **View**.
4. On the event detail page, click the **Audit Trail** tab.
5. You will see a table of every change made to this event, including the timestamp, the user who made the change, the field that was changed, the old value, the new value, and the reason. This log is identical to the audit trail for events originating from automated integrations.
6. The event also appears in all reporting views and invoice runs alongside automated events — there is no separate pipeline for manual events.
7. The **Origin** field on the detail page and in the list is set to **MANUAL**, providing traceability of how the event entered the system.

**Confirms requirement:** The Audit Trail tab on the event detail page shows a complete, filterable change history for manual events, identical to automated events, confirming audit and reporting integration.

**UI Status:** ✅ VISIBLE

## Orphaned UI Noticed

- **Save as Draft** — The Create Billing Event form includes a dedicated **Save as Draft** button that only requires Date and Customer to save. This reduces required fields on hover to just two. This is a UX affordance not explicitly called out in PD-283's ACs but is a natural extension of the manual creation flow. It is connected to billing event status lifecycle (PD-297) so it is not orphaned — just worth noting as an extra convenience feature.

- **Billing Event Templates** — The Create Billing Event page includes a template loader (dropdown) and a **Save as Template** button, plus a **Manage templates** button linking to `/billing-events/templates`. This feature allows reusable templates to be saved and applied to new events. It is referenced in the CreateBillingEventPage file linked to PD-283 but not explicitly described as an AC in PD-283. This may be covered by another requirement or is an additional convenience feature that currently lacks a dedicated PRD entry visible during this scan.
