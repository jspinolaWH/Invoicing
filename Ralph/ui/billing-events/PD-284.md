## Where to Find This Feature

This feature spans several areas of the app. The classification of billing events as Public Law or Private Law is visible in the Billing Events list and on each event detail page. The classification appears on invoice line items in the Invoices section and the Authority Invoice View. Configuration of the default classification for a customer lives in their Billing Profile, and classification rules are managed under Customers & Config.

---

## User Flows by Acceptance Criterion

---

### AC1 — View public/private law classification across the system

1. Click **Operations** in the top navigation bar.
2. In the left sidebar, click **Billing Events**.
3. You will see a table of billing events. Each row that has a legal classification assigned shows a coloured badge — a blue "Public" badge for public-law events and an orange "Private" badge for private-law events.
4. Click the **View** button on any billing event row.
5. On the event detail page, scroll to the **Details** tab (shown by default). Find the **Classification** field in the detail grid. It displays the raw classification value (e.g. `PUBLIC_LAW` or `PRIVATE_LAW`).
6. You can also navigate to **Invoices** in the top navigation bar, then click **Invoice List** in the sidebar, and click any invoice row.
7. On the invoice detail page, open the **Line Items** tab. Each line in the table has a **Classification** column showing a coloured badge (blue = Public Law, orange = Private Law).

**Confirms requirement:** The classification is visible at both the billing event level and the invoice line level, confirming that transactions are correctly classified and the classification is carried through to invoices.

**UI Status:** ✅ VISIBLE

---

### AC2 — Configure how public and private transactions are invoiced

1. Click **Customers & Config** in the top navigation bar.
2. In the left sidebar, click **Billing Profiles**.
3. You will see a list of customers. Click any customer row to open their billing profile.
4. On the billing profile page, scroll down to the **Reporting Defaults** section.
5. You will see a **Default Legal Classification** dropdown. Choose either **PUBLIC LAW** or **PRIVATE LAW** to set the customer's default classification for events that do not have one explicitly assigned.
6. Click **Save Billing Profile** to apply.

**Confirms requirement:** A customer-level default classification can be set, which drives how events are classified when no explicit classification is present. However, there is no dedicated toggle to choose whether public and private transactions should appear on the same invoice or on separate invoices — that is controlled indirectly via invoice run filters (see AC4 and AC7).

**UI Status:** ⚠️ PARTIAL

*What is missing:* There is no explicit "Combine public and private on one invoice" vs. "Separate into different invoices" toggle on the billing profile. The user must use invoice run filters as a workaround.

---

### AC3 — See legal classification identifier at the invoice line level (combined invoice)

1. Click **Invoices** in the top navigation bar.
2. In the left sidebar, click **Invoice List**.
3. You will see a table of all invoices. Click any invoice row to open the invoice detail page.
4. On the detail page, locate the **Line Items** section. The table shows columns for Description, Qty, Unit Price, VAT %, Net, Gross, Classification, Account, and Cost Center.
5. The **Classification** column displays a coloured badge for each line — blue "Public Law" for public-law charges and orange "Private Law" for private-law charges. A single invoice may contain lines with different classifications.

**Confirms requirement:** Even when public and private charges appear on the same invoice, each line carries a distinct classification badge, satisfying the requirement for a clear identifier at the invoice line level.

**UI Status:** ✅ VISIBLE

---

### AC4 — Separate public and private transactions into different invoices

1. Click **Invoices** in the top navigation bar.
2. In the left sidebar, click **Invoice Runs**.
3. On the Invoice Run configuration page, locate the filter form.
4. Find the **Service Responsibility** dropdown. Set it to either **Public Law** or **Private Law**.
5. Run the invoice run with this filter applied. Only events matching the selected service responsibility will be included in this run, producing invoices that contain only public-law or only private-law charges.
6. Repeat the process with the opposite filter to generate the complementary set of invoices.

**Confirms requirement:** By running separate invoice runs filtered by Service Responsibility (Public Law / Private Law), users can produce separate invoices for each classification. However, this is a manual process requiring two separate runs rather than a single automated "separate by classification" setting.

**UI Status:** ⚠️ PARTIAL

*What is missing:* There is no single configuration option that automatically splits invoices by legal classification. Users must run two invoice runs manually.

---

### AC5 — Customize invoice processing rules at the company level

1. Click **Customers & Config** in the top navigation bar.
2. In the left sidebar, click **Classification Rules**.
3. You will see a table of classification rules, each with a Priority, Label, Customer Type condition, Product Code condition, Region condition, Result Classification (PUBLIC_LAW or PRIVATE_LAW), and an Active flag.
4. Click **+ Add Rule** to create a new rule.
5. In the modal, set a priority number (lower = evaluated first), optionally constrain the rule to a specific Customer Type, Product Code, or Region, then choose the Result Classification.
6. Click **Save** to apply the rule.
7. The system will use these rules to automatically assign classifications to billing events that do not have an explicit classification.

**Confirms requirement:** Company-level classification rules can be configured to drive how events are classified. However, per-company settings for whether invoices are combined or separated by law type are not available — this is handled at the invoice run level.

**UI Status:** ⚠️ PARTIAL

*What is missing:* There is no company-level setting to define invoice grouping behavior (combine vs. separate public/private). Classification rules assign the law type but do not control invoice grouping strategy.

---

### AC6 — Distinguish transactions using metadata (for enforcement, VAT, and reporting)

1. Click **Operations** in the top navigation bar.
2. In the left sidebar, click **Billing Events**.
3. On any billing event row, click **View**.
4. On the event detail page (Details tab), you will see the **Classification** field which shows the legal classification of this event (PUBLIC_LAW or PRIVATE_LAW).
5. You will also see **VAT Details** (VAT rate code, effective rate, net, VAT and gross amounts), **Accounting Account**, and **Cost Center** fields — all of which are tied to the classification.
6. Navigate to **Invoices** and open any invoice detail. The line items table shows the Classification column alongside VAT %, Account, and Cost Center columns, confirming all metadata travels with the line.

**Confirms requirement:** Each billing event and invoice line carries classification metadata alongside VAT and financial account data, enabling correct enforcement routing and financial reporting.

**UI Status:** ✅ VISIBLE

---

### AC7 — User-defined invoice grouping (same or separate invoices)

1. Click **Invoices** in the top navigation bar.
2. In the left sidebar, click **Invoice Runs**.
3. In the filter form, use the **Service Responsibility** dropdown to filter to Public Law or Private Law before running.
4. To combine both on one invoice: leave the Service Responsibility filter blank and run. All qualifying events regardless of classification will be invoiced together, and each invoice line will carry its own classification badge.
5. To separate: run two separate runs, one filtered to Public Law and one to Private Law.

**Confirms requirement:** Users can choose — via the invoice run filter — whether to combine or separate public and private transactions. However, this is not a per-customer persistent configuration; it is a run-time decision at each invoice run.

**UI Status:** ⚠️ PARTIAL

*What is missing:* There is no stored per-customer or per-company preference for invoice grouping by legal classification. The decision is made ad hoc at each invoice run.

---

### AC8 — Legal classification at the line level on invoices

1. Click **Tools** in the top navigation bar.
2. In the left sidebar, click **Authority View**.
3. Optionally enter a Customer ID and date range, then click **Search**.
4. You will see a list of sent and completed invoices. Click any invoice row.
5. An expanded panel appears below the table showing the invoice's line items. The table includes a **Classification** column. Each line that has a legal classification shows a blue "PUBLIC_LAW" or grey "PRIVATE_LAW" badge.

**Confirms requirement:** The legal classification is visible at the individual invoice line level, both in the main invoice detail view and in the Authority Invoice View.

**UI Status:** ✅ VISIBLE

### AC9 — Automatic compliance with legislation in collection processes

There is no user-facing screen for this acceptance criterion. The system automatically assigns the PUBLIC_LAW or PRIVATE_LAW classification to billing events based on the classification rules configured under Customers & Config. This classification then determines which VAT rules and accounting accounts are applied. The logic runs in the backend classification engine whenever events are created or an invoice run is executed.

**UI Status:** ❌ NOT VISIBLE

*Why there is no UI:* This is fully automated backend behavior. The user configures rules (see AC5) and the system applies them automatically. No manual step is required or expected.

---

### AC10 — Integration with FINVOICE and external systems (classification exported)

1. Click **Invoices** in the top navigation bar.
2. In the left sidebar, click **Invoice List**.
3. Select one or more invoices using the checkboxes (only invoices with SENT or COMPLETED status can be selected).
4. When invoices are selected, a toolbar appears. Click **Set Text**.
5. In the modal, enter custom text. The label reads: "Custom text (shown on invoice and in FINVOICE)". Click **Apply Text**.
6. The classification data embedded in each invoice line (PUBLIC_LAW / PRIVATE_LAW) is automatically included in the FINVOICE export when the invoice is transmitted — this happens without additional user action.

**Confirms requirement:** The FINVOICE export includes classification metadata from invoice lines automatically. The "Set Text" function confirms the app is aware of FINVOICE as an output channel, but there is no screen where users can see or configure the specific FINVOICE classification field mapping.

**UI Status:** ⚠️ PARTIAL

*What is missing:* There is no page showing FINVOICE-specific output or confirming that the classification field is correctly mapped in the FINVOICE envelope. The export is backend-automatic; users cannot preview or verify the classification in the FINVOICE payload from the UI.

---

### AC11 — Customizable enforcement handling (separate enforcement authorities)

There is no dedicated screen for configuring enforcement routing. No "enforcement authority" configuration, no "collection process" routing settings, and no per-classification enforcement channel configuration were found in the frontend.

**UI Status:** ❌ NOT VISIBLE

*Why there is no UI:* Enforcement routing (whether a public-law invoice goes to a statutory enforcement authority vs. a private-law invoice goes through general debt collection) appears to be handled outside the invoicing application — likely in the downstream operator or ERP system. The invoicing app provides the classification metadata (PUBLIC_LAW / PRIVATE_LAW) on each invoice line, but the enforcement routing decision is not configurable within this UI.

---

## Backend-Only Aspects

- **AC9 — Automatic classification engine:** The system automatically evaluates classification rules against each billing event's customer type, product code, and region, and assigns the resulting PUBLIC_LAW or PRIVATE_LAW classification. Users configure the rules but do not manually trigger the engine; it runs automatically during event creation and invoice runs.
- **AC11 — Enforcement routing:** The legal classification is attached to invoice lines and exported in FINVOICE data, but the actual enforcement authority routing (which body handles collection for public-law vs. private-law invoices) is not managed within this application.

## Orphaned UI Noticed

- **Authority Invoice View classification column** — The Authority Invoice View page at Tools > Authority View renders a "Classification" column in the line items panel using inline styles rather than the shared `ClassificationBadge` component. It duplicates badge rendering logic. This is a minor implementation inconsistency but does not appear to be linked to a separate requirement beyond PD-284.
