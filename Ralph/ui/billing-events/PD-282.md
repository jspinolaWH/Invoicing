## Where to Find This Feature

This feature lives under the **Customers & Config** tab. Access it by clicking any customer row on the Billing Profiles page, then scrolling down to the **E-Invoice Address** section of that customer's profile — which only appears when the delivery method is set to **E INVOICE**.

## User Flows by Acceptance Criterion

---

### AC1 — Manually adding an e-invoicing address

1. Click **Customers & Config** in the top navigation bar.
2. In the left sidebar, click **Billing Profiles**.
3. You will see a table of customers. Click any customer row to open their billing profile.
4. On the billing profile page, locate the **Delivery Method** field. Change it to **E INVOICE** using the dropdown.
5. Scroll down below the **Save Billing Profile** button. A new section titled **E-Invoice Address** will appear.
6. In the **E-Invoice Address** field, type the customer's e-invoice address (up to 35 characters).
7. Optionally fill in the **Operator Code** field (e.g. OKOYFIHH).
8. Click **Save E-Invoice Address**.
9. A green success message reading "E-invoice address saved." will appear at the top of that section.

**Confirms requirement:** The user can type a new e-invoice address into the form and save it; the address is then stored on the customer record.

**UI Status:** ✅ VISIBLE

---

### AC2 — Override or modify e-invoice address and prevent integration from replacing it

1. Click **Customers & Config** in the top navigation bar.
2. In the left sidebar, click **Billing Profiles**.
3. Click any customer row whose delivery method is already **E INVOICE**, or change it as described in AC1.
4. Scroll to the **E-Invoice Address** section.
5. Edit the **E-Invoice Address** field with the correct/overridden address.
6. Tick the **Manually locked (prevent operator updates)** checkbox.
7. A **Lock Reason** field appears below the checkbox. Optionally type a reason explaining why the address is locked.
8. Click **Save E-Invoice Address**.
9. A green success message "E-invoice address saved." confirms the lock is stored.

**Confirms requirement:** The lock checkbox explicitly prevents the integration from automatically replacing the saved e-invoice address, and the user can set a reason for the lock.

**UI Status:** ✅ VISIBLE

---

### AC3 — E-invoicing addresses retrieved automatically via integration (no user approval required)

This criterion describes a fully automated backend process. When the integration pushes an e-invoice address for a customer, the backend stores it directly without requiring the user to approve or confirm anything in the UI. Users have no screen to visit for this flow — it happens silently in the background.

The e-invoice integration itself is covered in requirement 3.11.24.

**Confirms requirement:** N/A — no user action is needed.

**UI Status:** ❌ NOT VISIBLE

**Why no UI:** The automatic retrieval from the integration is a backend-only operation. There is no approval workflow or notification surface in the frontend for this step. The result of the automatic retrieval is visible on the Billing Profile page (the address will appear pre-populated if the integration has run), but the retrieval act itself is invisible.

---

### AC4 — E-invoicing addresses can be edited and added manually

1. Click **Customers & Config** in the top navigation bar.
2. In the left sidebar, click **Billing Profiles**.
3. Click any customer row to open the billing profile.
4. Ensure the **Delivery Method** is set to **E INVOICE**. If not, change it and click **Save Billing Profile** first.
5. Scroll down to the **E-Invoice Address** section.
6. The **E-Invoice Address** field is a free-text input. Clear it and type a new address to add, or edit the existing value to update it.
7. Click **Save E-Invoice Address**.
8. You will see the green "E-invoice address saved." success message.

**Confirms requirement:** The input field accepts both new entries and modifications to an existing address, covering both "add" and "edit" scenarios.

**UI Status:** ✅ VISIBLE

---

### AC5 — Checkbox setting to prevent integration from overwriting the e-invoice address

1. Click **Customers & Config** in the top navigation bar.
2. In the left sidebar, click **Billing Profiles**.
3. Click a customer row to open their billing profile.
4. Ensure the **Delivery Method** is **E INVOICE** so the **E-Invoice Address** section is visible.
5. Scroll to the **E-Invoice Address** section.
6. Tick the **Manually locked (prevent operator updates)** checkbox. This is the field described in the requirement as "a checkbox indicating that this field must not be overwritten by the integration."
7. When the checkbox is ticked, a **Lock Reason** text field appears beneath it. Enter an optional explanation.
8. Click **Save E-Invoice Address** to persist the lock.
9. The green success message confirms the setting is saved. From this point on, integration updates will not overwrite the stored e-invoice address.

**Confirms requirement:** The checkbox labelled "Manually locked (prevent operator updates)" is exactly the field described in the requirement. When checked and saved, the backend respects this flag and skips integration-driven overwrites.

**UI Status:** ✅ VISIBLE

## Backend-Only Aspects

There is no approval step or notification UI. This is intentional — the requirement states "without requiring user approval." The address silently populates in the customer's billing profile and becomes visible when a user next opens that profile page.
