## Where to Find This Feature

This feature is spread across the **Operations** tab. The main status summary view lives on the **Billing Events** list page (showing a strip of status counts at the top). Customer-specific status information is found inside a customer's **Billing Profile** page under the Customers & Config tab.

---

## User Flows by Acceptance Criterion

---

### AC1 — Status of invoices visible at the event level

1. Click **Operations** in the top navigation bar.
2. In the left sidebar, click **Billing Events**.
3. You will see a table of all billing events. Each row has a **Status** column showing a coloured badge (e.g. "In Progress", "Sent", "Completed", "Error").
4. Click the **View** button on any event row.
5. On the Event Detail page, you will see the event's current status displayed as a coloured badge directly next to the event title at the top of the page.

**Confirms requirement:** The status badge on both the list row and the detail page header confirms that the invoice/processing status is visible at the individual event level.

**UI Status:** ✅ VISIBLE

---

### AC2 — Default status categories (In Progress, Sent, Error, Completed) are present

1. Click **Operations** in the top navigation bar.
2. In the left sidebar, click **Billing Events**.
3. Near the top of the page, you will see a row of clickable status summary tiles. Each tile shows one status label and a count. The tiles include: **Draft**, **Pending Transfer**, **In Progress**, **For Correction**, **Sent**, **Completed**, **Error**, **Validation Failed**, and **Excluded**.
4. You can also see these statuses in the filter bar — open the **Status** dropdown and you will see all statuses listed as filter options.

**Confirms requirement:** All four default categories specified in the requirement (In Progress, Sent, Error, Completed) are present as distinct status values with coloured badges and summary counts.

**UI Status:** ✅ VISIBLE

---

### AC3 — In Progress: events recorded but invoice not yet created

1. Click **Operations** in the top navigation bar.
2. In the left sidebar, click **Billing Events**.
3. In the filter bar, open the **Status** dropdown and select **IN_PROGRESS**, then click **Apply**.
4. The table will show only events with the yellow/amber "In Progress" badge in the Status column.
5. Alternatively, click the **In Progress** tile in the status strip at the top to filter automatically.

**Confirms requirement:** The "In Progress" status badge is clearly visible and filterable, representing events that have been recorded but not yet invoiced.

**UI Status:** ✅ VISIBLE

---

### AC4 — Sent: invoice generated but data not yet confirmed by external system

1. Click **Operations** in the top navigation bar.
2. In the left sidebar, click **Billing Events**.
3. Click the **Sent** tile in the status strip, or open the **Status** dropdown and select **SENT**, then click **Apply**.
4. The table will show events with the blue "Sent" badge — these have had invoice data generated and sent to the invoicing system but not yet confirmed as received.

**Confirms requirement:** The "Sent" status badge is visible, filterable, and distinguishable from "Completed", confirming the intermediate state between invoice generation and external system confirmation.

**UI Status:** ✅ VISIBLE

---

### AC5 — Error: incorrect data or transfer errors

1. Click **Operations** in the top navigation bar.
2. In the left sidebar, click **Billing Events**.
3. Click the **Error** tile in the status strip, or select **ERROR** in the status filter dropdown and click **Apply**.
4. You will see events with a red "Error" badge. Click the **View** button on any error event.
5. On the Event Detail page, click the **Status** tab. If a transmission error reason is recorded, a red panel will appear showing the error message.

**Confirms requirement:** The "Error" status is visible in the list, filterable, and the detail page's Status tab shows the specific transmission error reason when available.

**UI Status:** ✅ VISIBLE

---

### AC6 — Completed: invoice created and data confirmed by invoicing system

1. Click **Operations** in the top navigation bar.
2. In the left sidebar, click **Billing Events**.
3. Click the **Completed** tile in the status strip, or select **COMPLETED** in the status filter dropdown.
4. Events with the green "Completed" badge are shown — these have had invoice data successfully sent to and confirmed by the invoicing system.

**Confirms requirement:** The "Completed" status badge is visually distinct (green) and clearly signals the end of the invoicing workflow for a billing event.

**UI Status:** ✅ VISIBLE

---

### AC7 — Sent events can be updated; data corrected and re-sent; then marked Completed

1. Click **Operations** in the top navigation bar.
2. In the left sidebar, click **Billing Events**.
3. Find an event with **Sent** status and click **View**.
4. On the Event Detail page, click the **Status** tab.
5. You will see the current status displayed and a button labelled **Mark as Completed**. This allows a user to manually confirm that the external system has received and accepted the data.
6. If corrections are needed before completion, a user with the appropriate role can use the **Credit & Transfer** button (visible on Sent events) to issue a corrected event.

**Confirms requirement:** The Status tab on a Sent event shows the transition control to Completed, and the Credit & Transfer capability is available — confirming the ability to update/correct and re-submit before final completion.

**UI Status:** ✅ VISIBLE

---

### AC8 — Events have process-based statuses that update automatically as workflow progresses

1. Click **Operations** in the top navigation bar.
2. In the left sidebar, click **Billing Events**.
3. Click **View** on any billing event.
4. On the Event Detail page, click the **Audit Trail** tab.
5. You will see a table of all changes made to this event. The **Field** column will show "Status" for any automated or manual status transitions. The **Previous Value** and **New Value** columns show how the status changed over time.

**Confirms requirement:** The Audit Trail tab records every status change (automatic and manual) so users can see the full status history of any event, confirming that process-based status tracking is active.

**UI Status:** ✅ VISIBLE

---

### AC9 — Event statuses visible in summary view AND customer-specific view

**Summary view:**
1. Click **Operations** in the top navigation bar.
2. In the left sidebar, click **Billing Events**.
3. The status strip at the top shows counts for every status across all events — this is the global summary view.

**Customer-specific view:**
1. Click **Customers & Config** in the top navigation bar.
2. In the left sidebar, click **Billing Profiles**.
3. Click any customer row to open their Billing Profile.
4. Scroll down to the **Billing Events** section at the bottom of the profile page.
5. You will see a row of status summary tiles showing how many events this specific customer has in each status (Draft, Pending Transfer, In Progress, Sent, Completed, Error).
6. Below the summary tiles, a table shows the customer's most recent billing events with a Status badge on each row.

**Confirms requirement:** Statuses are visible both in the global summary (Billing Events page stat strip) and in customer-specific detail (Billing Profile page Billing Events section).

**UI Status:** ✅ VISIBLE

---

### AC10 — Events editable until marked as Completed and invoice successfully transferred

1. Click **Operations** in the top navigation bar.
2. In the left sidebar, click **Billing Events**.
3. Click **View** on a billing event.
4. Observe the **Edit** button in the top-right action area. This button is enabled only for events with status: Draft, In Progress, For Correction, or Error. It is disabled (greyed out) for events with status Sent, Completed, or Pending Transfer.
5. Hover over the disabled Edit button — a tooltip will appear saying "Event is [STATUS] and cannot be edited".

**Confirms requirement:** The Edit button's conditional availability enforces the rule that events can only be edited before they are fully completed and transferred.

**UI Status:** ✅ VISIBLE

---

### AC11 — Incorrect event data correctable after invoicing run if data not yet received by external system

1. Click **Operations** in the top navigation bar.
2. In the left sidebar, click **Billing Events**.
3. Find an event with **Error** status (data transfer failed) or **For Correction** status, and click **View**.
4. For an **Error** event: the **Edit** button is enabled, allowing the user to correct the data and resubmit.
5. For a **For Correction** event: a purple banner at the top explains that the event was manually edited and is awaiting approval from an INVOICING user. The **Approve Correction** button appears in the header for authorised users.
6. On events in **Sent** status (data sent but external system has not yet confirmed): the **Credit & Transfer** button allows issuing a corrective credit event and a new corrected transfer event.

**Confirms requirement:** Multiple correction pathways are visible in the UI — direct edit for Error/In Progress events, the For Correction approval flow for post-edit review, and the Credit & Transfer mechanism for already-sent events.

**UI Status:** ✅ VISIBLE
