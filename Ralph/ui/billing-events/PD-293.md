## Where to Find This Feature

This feature is spread across two areas of the app. The primary place to apply invoice batch filters is the **Invoice Runs** page under the **Invoices** tab — this is where all selection criteria (municipality, minimum amount, period, customer type, service type, location, billing frequency, and service responsibility) are entered before each run. Persistent exclusion rules that survive across runs are configured under **Restrictions** in the **Operations** tab.

## User Flows by Acceptance Criterion

---

### AC1 — Filter invoice run by municipality

1. Click **Invoices** in the top navigation bar.
2. In the left sidebar, click **Invoice Runs**.
3. You will see a page titled "Invoice Run" with two mode options at the top: **Real Run** and **Simulate First**.
4. Below the mode selector you will see a form. The first field in the top row is labelled **Municipality**.
5. Type the municipality name (e.g. "Tampere") into the Municipality field.
6. Set any other desired filters and click **Start Real Run** (or **Run Simulation** to preview first).
7. The system will include only billing events from the specified municipality in this run.

**Confirms requirement:** The Municipality filter field is present on the Invoice Runs form, letting the user restrict which events are invoiced by municipality.

**UI Status:** ✅ VISIBLE

### AC2 — Minimum monetary threshold below which invoices are not generated

1. Click **Invoices** in the top navigation bar.
2. In the left sidebar, click **Invoice Runs**.
3. In the filter form, locate the **Min Amount (€)** field in the top row (next to Municipality).
4. Enter a monetary value (e.g. "10.00"). The system will skip generating invoices for totals below this amount.
5. Alternatively, to set a persistent minimum amount rule that applies across all runs: click **Operations** in the top navigation bar, then click **Restrictions** in the left sidebar.
6. On the Billing Restrictions page, click **+ Add Restriction**.
7. In the modal, set **Billing Type** to CYCLE_BASED and enter the desired amount in the **Min Amount (€)** field.
8. Click **Save**. The restriction will now apply to all future invoicing runs that match its other criteria.

**Confirms requirement:** Both a per-run minimum amount field and a persistent Billing Restriction with a Min Amount setting are available.

**UI Status:** ✅ VISIBLE

### AC9 — Only a subset of events linked to a customer can be invoiced

1. Click **Operations** in the top navigation bar.
2. In the left sidebar, click **Restrictions**.
3. The Billing Restrictions page shows all configured exclusion rules. Each rule targets a subset of events (by service type, municipality, customer type, minimum amount, etc.).
4. Click **+ Add Restriction** to create a new rule.
5. Configure the conditions (e.g. Service Type = SEPTIC_TANK_EMPTYING, Customer Type = BUSINESS) and save.
6. During the next invoice run, the system will apply these rules and invoice only the events that pass all configured restrictions — excluding the rest.

**Confirms requirement:** Billing Restrictions allow defining conditions that determine which events are invoiced, effectively restricting invoicing to a subset of a customer's events.

**UI Status:** ✅ VISIBLE

### AC10 — User sets conditions for event exclusion (customisable print rules)

1. Click **Operations** in the top navigation bar.
2. In the left sidebar, click **Restrictions**.
3. Click **+ Add Restriction**.
4. In the modal, fill in any combination of: Service Type, Municipality, Customer Type, Min Amount, Period, Service Responsibility, and Billing Type (CYCLE_BASED or IMMEDIATE).
5. Check the **Active** checkbox and click **Save**.
6. The restriction now defines conditions under which events are excluded from invoicing. You can edit existing restrictions by clicking **Edit** on any row, or deactivate them with **Deactivate**.

**Confirms requirement:** Users can create, edit, and deactivate billing restriction rules that specify exactly which events should be excluded from invoice generation.

**UI Status:** ✅ VISIBLE

### AC12 — Minimum invoice amount threshold with carry-over behaviour

1. On the **Invoice Runs** page (Invoices → Invoice Runs), enter a value in the **Min Amount (€)** field to skip invoices below that threshold for this specific run.
2. For persistent carry-over behaviour, go to **Operations** → **Restrictions** → **+ Add Restriction** and set the **Min Amount (€)** field. Events that fall below this amount will not be invoiced in any matching run.
3. A dedicated minimum fee configuration also exists: click **Master Data** in the top navigation bar, then **Min. Fees** in the left sidebar. Here you can configure the exact threshold per customer type and period, with adjustment product codes.

**Confirms requirement:** Minimum invoice amount filtering is available at both the run level (Invoice Runs) and at the persistent rule level (Billing Restrictions and Min. Fees config).

**UI Status:** ✅ VISIBLE

### AC19 — Additional configurable criteria; visible selection fields defined

The Invoice Runs filter form has a fixed set of fields (Municipality, Min Amount, Period From/To, Service Responsibility, Location, Customer Type, Service / Product Code, Billing Frequency, Invoice Template, Invoice Number Series). There is no UI for an administrator to add new custom filter fields or configure which fields are shown in the invoice printing view — the set of available criteria is hardcoded in the form.

**Confirms requirement:** Most criteria are present, but the requirement states "other fields can also be configured as criteria" and "fields to be visible need to be defined" — there is no admin screen for dynamically adding or toggling which selection criteria appear.

**UI Status:** ⚠️ PARTIAL — The fixed set of filter fields satisfies the majority of named criteria, but there is no UI for dynamically configuring or extending the set of visible selection criteria beyond what is hardcoded in the Invoice Runs form.
