## Where to Find This Feature

This feature is spread across two areas of the app. The master data side (defining and managing number series and invoice templates) lives under **Master Data**. The runtime side (selecting a series or template when running invoices) lives under **Invoices > Invoice Runs**.

## User Flows by Acceptance Criterion

---

### AC1 — Define and apply an invoice number format

1. Click **Master Data** in the top navigation bar.
2. In the left sidebar, click **Number Series**.
3. You will see a table listing all existing invoice number series. Each row shows the series Name, Prefix, Category, Format Pattern, Current Counter, and Released count.
4. Click **+ Add Series** in the top-right corner.
5. A modal dialog opens. Fill in:
   - **Series Name** (required) — a unique identifier for this series, e.g. "MAIN_2026"
   - **Prefix** (required) — a short code, e.g. "INV"
   - **Category** — optionally select SLUDGE, CREDIT, PUBLIC, PRIVATE, or ANNUAL
   - **Format Pattern** (required) — define the structure using placeholders such as `{PREFIX}-{YEAR}-{COUNTER:06d}`
6. As you type, a live **Preview** appears below the Format Pattern field showing what a generated number will look like (e.g. `INV-2026-000001`).
7. Click **Save**. The new series appears in the table with a counter starting at 0.

**Confirms requirement:** The user can define any format pattern, and the system will use it to generate all subsequent invoice numbers from that series.

**UI Status:** ✅ VISIBLE

---

### AC2 — Select number series manually or from pre-defined series during invoice creation

1. Click **Invoices** in the top navigation bar.
2. In the left sidebar, click **Invoice Runs**.
3. The Invoice Run configuration page opens.
4. Scroll to the **Invoice Number Series** row in the filter form.
5. Use the **Series Category** dropdown to narrow the list to a specific category (e.g. SLUDGE, CREDIT, PUBLIC, PRIVATE, ANNUAL).
6. Use the **Invoice Number Series** dropdown to select the specific series to use for this run. The dropdown shows the series name, prefix, and category. Choosing "— Use default series —" leaves the system to pick automatically.

**Confirms requirement:** The user can either leave the series unset (automatic/default) or explicitly choose from any pre-defined numbered series before starting the run.

**UI Status:** ✅ VISIBLE

### AC3 — Define invoice number structure in an invoice template; numbers always unique

1. Click **Master Data** in the top navigation bar.
2. In the left sidebar, click **Overview** (the Master Data hub page).
3. You will see a grid of cards. Click the **Invoice Templates** card.
4. The Invoice Templates page opens, showing a table with columns: Name, Code, Number Series, Created By, and Actions.
5. Click **+ Add Template** in the top-right corner.
6. A modal dialog opens. Fill in:
   - **Name** (required) — a human-readable label for the template
   - **Code** (required) — a short machine code, e.g. "WASTE_STANDARD"
   - **Number Series** (optional) — select one of the pre-defined number series from the dropdown; the linked series defines the format and counter for invoices using this template
7. Click **Save**. The template appears in the table showing the linked number series name.

The system uses atomic counter assignment to ensure each number is issued exactly once — duplicate numbers are structurally prevented at the series level.

**Confirms requirement:** Each invoice template carries a number series definition, and the atomic counter mechanism guarantees uniqueness for every invoice generated using that template.

**UI Status:** ✅ VISIBLE

### AC4 — Full control over defining and managing invoice number series

1. Click **Master Data** in the top navigation bar.
2. In the left sidebar, click **Number Series**.
3. You will see the full list of all defined invoice number series.
4. To **add** a new series: click **+ Add Series**, fill in Name, Prefix, Category, and Format Pattern, then click **Save**.
5. To **edit** an existing series: click the **Edit** button on any row. The same modal opens pre-filled with the existing values. Change any field and click **Save**.
6. To **delete** a series: click the **Delete** button on any row. A confirmation dialog asks "Delete series [name]? This cannot be undone." Clicking OK removes the series (deletion is blocked if the series is currently in use).
7. To **test** number assignment: click the **Test Assign** button on any row. A panel expands below the row showing a Simulation Mode checkbox (ticked by default — counter is NOT incremented). Click **Assign Number** to preview the next number that would be generated. Untick Simulation Mode and confirm to permanently increment the real counter.

**Confirms requirement:** Users have complete CRUD control over every number series and can test assignment safely before committing to real counter increments.

**UI Status:** ✅ VISIBLE

---

### AC5 — System prevents duplicate and invalid numbering

This is partly enforced in the backend and partly visible through the UI:

- The **Test Assign** panel on the Invoice Number Series page uses atomic counter logic. The UI surface for this is the **Test Assign** button on each series row (see AC4 step 7).
- When a user attempts to delete a series that is already referenced by invoices, the system shows an error message: "Delete failed. Series may be in use."
- When saving a series with a duplicate name, the system shows: "Save failed. The series name may already exist."
- The format pattern preview in the Add/Edit modal makes it easy to spot invalid patterns before saving.

**Confirms requirement:** The user sees error feedback when attempting actions that would cause duplicate or invalid numbering. The atomic counter backend mechanism is surfaced through the Test Assign panel.

**UI Status:** ✅ VISIBLE

### AC6 — Select series based on pre-defined categories

1. Click **Invoices** in the top navigation bar.
2. In the left sidebar, click **Invoice Runs**.
3. Scroll to the **Series Category** dropdown in the filter form.
4. Select a category (e.g. SLUDGE, CREDIT, PUBLIC, PRIVATE, ANNUAL). The **Invoice Number Series** dropdown immediately filters to show only series belonging to that category.
5. Select the desired series from the filtered list.

**Confirms requirement:** Users can narrow their series selection by category, matching the pre-defined categories (SLUDGE, CREDIT, PUBLIC, PRIVATE, ANNUAL) that were defined when each series was created.

**UI Status:** ✅ VISIBLE

---

### AC7 — Select invoice template with number series defined; series applied to that invoicing type

1. Click **Invoices** in the top navigation bar.
2. In the left sidebar, click **Invoice Runs**.
3. In the filter form, locate the **Invoice Template** dropdown.
4. Select a template from the list. Each option shows the template name, its code in parentheses, and — if it has a linked number series — the series name separated by a dot (e.g. "Standard Waste Invoice (WASTE_STANDARD) · MAIN_2026").
5. When a template is selected, the **Invoice Number Series** field is automatically pre-filled with the series linked to that template. The user can still override this manually if needed.
6. Complete the remaining filter fields and click **Start Real Run** (or **Run Simulation** to preview first).

**Confirms requirement:** Selecting an invoice template automatically carries in its associated number series, ensuring the correct numbering type is applied to the invoicing run without requiring manual series selection.

**UI Status:** ✅ VISIBLE
