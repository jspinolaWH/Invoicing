## Where to Find This Feature

This feature is spread across several areas of the app. Templates (with their codes) are defined under **Master Data**, then assigned at the customer level in **Customers & Config**, at the property level inside a customer's billing profile, at the contract level also inside the billing profile, and at the batch level when starting an **Invoice Run** under the **Invoices** tab.

## User Flows by Acceptance Criterion

---

### AC1 — Set an invoice template for a customer

1. Click **Customers & Config** in the top navigation bar.
2. In the left sidebar, click **Billing Profiles**.
3. You will see a table of customers. Click on any customer row to open their billing profile.
4. The page opens with a **Profile** tab active. Scroll down to the **Invoice Template** field — it is a dropdown that lists all defined templates.
5. Select the desired template from the dropdown (each option shows the template name and its code, e.g. "Standard Waste Invoice (WASTE_STANDARD)").
6. Click **Save Billing Profile**.
7. You will see a green success message confirming the profile was saved.

**Confirms requirement:** The customer's billing profile now carries the selected invoice template, which will be used when generating invoices for that customer.

**UI Status:** ✅ VISIBLE

---

### AC2 — Set an invoice template for a property/site

1. Click **Customers & Config** in the top navigation bar.
2. In the left sidebar, click **Billing Profiles**.
3. Click on any customer row to open their billing profile.
4. Scroll down to the **Billing Events** section or look for property links. On the billing profile page, individual properties are linked from within the customer's data.

   Alternatively, navigate directly: the property detail page is reached from a customer's billing profile by clicking a property identifier link.

5. On the Property Detail page, scroll down to the **Invoice Template** section.
6. A dropdown is shown labelled "Invoice Template" with an option "— No template override —" plus all available templates.
7. Select the desired template and click **Save**.
8. A green "Template saved." message appears below the section heading.

**Confirms requirement:** The property now has its own invoice template override, separate from the customer-level default.

**UI Status:** ✅ VISIBLE

---

### AC3 — Set an invoice template for a contract

1. Click **Customers & Config** in the top navigation bar.
2. In the left sidebar, click **Billing Profiles**.
3. Click on any customer row to open their billing profile.
4. The **Profile** tab is active. Scroll down past the billing address section.
5. If the customer has contracts, a **Contracts** section appears with a table showing columns: Contract, Template, Work Site, Start Date, End Date.
6. In the **Template** column for any contract row, a dropdown lets you pick an invoice template from the list.
7. After selecting a template from the dropdown, click **Save** in that row.
8. The row updates to show the new template assignment.

**Confirms requirement:** Each contract can independently override the invoice template, allowing different templates per contract for the same customer.

**UI Status:** ✅ VISIBLE

---

### AC4 — Set an invoice template during an invoicing run

1. Click **Invoices** in the top navigation bar.
2. In the left sidebar, click **Invoice Runs**.
3. The Invoice Run configuration page opens. You see options for Real Run or Simulate First at the top.
4. In the filter form below, scroll to the **Invoice Template** field — a dropdown lists all defined templates (showing name, code, and linked number series).
5. Select the desired template. Selecting a template automatically pre-fills the **Invoice Number Series** field with the series linked to that template (though you can still change it manually).
6. Complete any other filters and click **Start Run** (or **Simulate**).
7. The run is created and you are taken to the run detail page.

**Confirms requirement:** An operator can choose a specific invoice template for an entire batch run, overriding any customer-level or property-level defaults for that run.

**UI Status:** ✅ VISIBLE

### AC5 — WasteHero does not create/manage templates; sends template info as FINVOICE data

The UI surface for template management is the **Invoice Templates** page under Master Data, which allows defining template records with a **Name** and a **Code**. The code is the identifier sent to the external invoicing system. The actual transmission of this code as part of the FINVOICE payload happens automatically in the backend when an invoice run is processed — there is no user-facing screen that shows the FINVOICE XML or confirms the template code was embedded in the outgoing data.

**To view the template code registry:**
1. Click **Master Data** in the top navigation bar.
2. You will see the Master Data hub. Click the **Invoice Templates** card (or navigate directly via the hub tiles).
3. A table shows all defined templates with columns: Name, Code, Number Series, Created By.
4. Click **+ Add Template** to create a new one; fill in the Name and Code fields.
5. The code (e.g. "WASTE_STANDARD") is what gets transmitted to the external invoicing system.

**Confirms requirement:** Partially — users can see and manage template codes in the UI. The actual FINVOICE data transmission is backend-only with no visible confirmation screen.

**UI Status:** ⚠️ PARTIAL

**What is missing:** There is no UI surface showing that the template code was included in the outgoing FINVOICE data, and no confirmation or log that shows what was actually transmitted to the external invoicing system. The requirement states WasteHero "will not create or manage invoice templates" — yet the app does have an Invoice Templates management page. This may be intentional (managing the code registry) but the separation between the WasteHero code registry and the external system's actual templates is not explained anywhere in the UI.

---

### AC6 — Template setting sent to external system using template codes

This acceptance criterion is satisfied automatically in the backend as part of invoice generation. When a template is selected (at customer, property, contract, or run level), the system resolves the template's code and includes it in the FINVOICE data sent to the external invoicing system. There is no dedicated UI screen showing this transmission.

**To observe the template code assignment at each level:**
- At the customer level: the template code is visible on the Billing Profile page (dropdown shows "Name (CODE)").
- At the property level: the template code is visible on the Property Detail page (dropdown shows "Name (CODE)").
- At the run level: the template code is visible in the Invoice Run filter form.
- At the contract level: the template code is visible in the Contracts table on the Billing Profile page.

**Confirms requirement:** Partially — template codes are clearly displayed in the selection dropdowns at all levels; the actual outbound FINVOICE transmission is handled by the backend with no UI confirmation.

**UI Status:** ⚠️ PARTIAL

**What is missing:** No UI screen confirms that the template code was successfully embedded in the FINVOICE payload sent to the external system. A transmission log or a "sent data preview" would complete this criterion visually.

---

## Backend-Only Aspects

- **FINVOICE data embedding:** When an invoice run executes, the backend resolves the effective template code (from the run, or from the contract, property, or customer — in priority order) and embeds it in the FINVOICE payload sent to the external invoicing system. This resolution and transmission is fully automatic with no user-facing screen.
- **Template code priority resolution:** The backend applies a precedence rule (run-level > contract-level > property-level > customer-level) to determine which template code to use for each invoice. This logic is invisible in the UI.

## Orphaned UI Noticed

- **Invoice Templates management page** (`/master-data/invoice-templates`) — This page allows creating, editing, and deleting invoice template records with a name, code, and linked number series. It is reachable from the Master Data hub but is not listed in the sidebar. It is linked to PD-307 via the `RELATED_TASKS` array in the file, but the requirement states WasteHero "will not create or manage invoice templates" — yet this page does exactly that. The purpose seems to be managing the code registry (the mapping between WasteHero template codes and external system templates), which is a valid interpretation, but the disconnect between the requirement text and the UI capability is worth flagging.
