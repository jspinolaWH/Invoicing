## Where to Find This Feature

This feature lives primarily under the **Invoices** tab. The batch PDF attachment is configured on the **Invoice Runs** page before starting a real run, and the resulting FINVOICE XML (including AttachmentDetails) can be inspected on any individual **Invoice Detail** page.

## User Flows by Acceptance Criterion

---

### AC1 — Single PDF attached to entire invoice batch via FINVOICE integration

1. Click **Invoices** in the top navigation bar.
2. In the left sidebar, click **Invoice Runs**.
3. You will see the Invoice Run configuration page with a mode selector at the top ("Real Run" and "Simulate First").
4. Make sure **Real Run** is selected — the batch attachment section only appears for real runs.
5. Below the filter form, you will see a panel titled **Batch PDF Attachment (optional)**.
6. In the **Attachment Identifier** field, type the unique identifier that the external invoicing broker assigned to the PDF file (for example, `FI-ATT-2026-001`). This identifier is how the invoicing service links the correct PDF to every invoice in the batch.
7. Optionally fill in **Filename**, **MIME Type**, and **Security Class**.
8. Click **Start Real Run**.

**Confirms requirement:** The system sends the batch attachment identifier to the backend alongside the run request, and the backend includes the `AttachmentDetails` section (with `AttachmentIdentifier`) in every FINVOICE message generated for this batch.

**UI Status:** ✅ VISIBLE

---

### AC2 — Users define a PDF attachment when creating a batch; AttachmentIdentifier included

1. Click **Invoices** in the top navigation bar.
2. In the left sidebar, click **Invoice Runs**.
3. Select **Real Run** mode.
4. Scroll down to the **Batch PDF Attachment (optional)** panel.
5. You will see four input fields: **Attachment Identifier**, **Filename**, **MIME Type** (pre-filled with `application/pdf`), and **Security Class**.
6. Enter the broker-assigned identifier in **Attachment Identifier** — this is the value that will appear as `AttachmentIdentifier` inside the `AttachmentDetails` block of each FINVOICE message.
7. Click **Start Real Run**.

**Confirms requirement:** The user can enter the AttachmentIdentifier before starting the batch, and the system includes it in the FINVOICE messages for all invoices in the run.

**UI Status:** ✅ VISIBLE

---

### AC3 — Attachment references correctly formatted and included in FINVOICE message

1. Click **Invoices** in the top navigation bar.
2. In the left sidebar, click **Invoice List**.
3. Click any invoice row to open the Invoice Detail page.
4. Scroll down to the **Attachments** section.
5. You will see a button labelled **↓ FINVOICE XML** in the top-right corner of the Attachments panel.
6. Click **↓ FINVOICE XML**. The browser will download an XML file for this invoice.
7. Open the downloaded XML file in any text editor and look for the `AttachmentDetails` block, which should contain the `AttachmentIdentifier` that was set during the invoice run.

**Confirms requirement:** The FINVOICE XML file is available for download, allowing a user to verify that the attachment reference is correctly formatted and present. However, the UI does not display the XML inline or show a parsed preview of the `AttachmentDetails` content — the user must inspect the raw XML manually.

**UI Status:** ⚠️ PARTIAL

**What is missing:** The UI provides a raw XML download but does not display a human-readable summary of the `AttachmentDetails` contents (e.g., the identifier, filename, and MIME type) on the invoice detail page. A tester cannot confirm correct formatting without opening the XML file externally.

### AC4 — Invoice data generation ensures invoicing service can associate the attachment

1. Click **Invoices** in the top navigation bar.
2. In the left sidebar, click **Invoice Runs**.
3. Select **Real Run** mode.
4. Fill in the desired filters (period, municipality, etc.) and enter the broker-assigned identifier in the **Batch PDF Attachment** panel.
5. Click **Start Real Run**.
6. The system navigates automatically to the Invoice Run Detail page, which shows the run status badge ("Running", then "Completed") and a summary card with total invoices generated and total amount.

**Confirms requirement:** The run creates invoices with the attachment identifier embedded in the generated FINVOICE data, enabling the external invoicing service to associate the file with the correct invoices.

**UI Status:** ✅ VISIBLE

### AC5 — Attachment reference in FINVOICE message (AttachmentDetails with AttachmentIdentifier)

1. Follow the same steps as AC3 above to download the FINVOICE XML from any invoice generated during the batch run.
2. The XML contains the `AttachmentDetails` section with the `AttachmentIdentifier` value that was entered at run time.

**Confirms requirement:** The FINVOICE XML download confirms that the `AttachmentDetails` section exists. The UI does not surface this data in a parsed, human-readable format on-screen.

**UI Status:** ⚠️ PARTIAL

**What is missing:** There is no on-screen display of `AttachmentDetails` metadata on the Invoice Detail page or the Invoice Run Detail page. Users must download and inspect the raw XML.

### AC6 — Correct attachment linking via structured metadata to the invoicing service

1. Click **Invoices** in the top navigation bar.
2. In the left sidebar, click **Invoice Runs**.
3. Select **Real Run** mode.
4. Enter the four metadata fields in the **Batch PDF Attachment (optional)** panel: **Attachment Identifier**, **Filename**, **MIME Type**, and **Security Class**.
5. Click **Start Real Run**.
6. The system sends this structured metadata to the backend, which embeds it in the FINVOICE messages so the invoicing service can correctly link the file to the batch.

**Confirms requirement:** All four metadata fields (identifier, filename, MIME type, security class) are available for input in the UI and are sent with the run.

**UI Status:** ✅ VISIBLE

### AC7 — Integration with external invoicing services for attachment handling

1. Click **Invoices** in the top navigation bar.
2. In the left sidebar, click **Invoice List**.
3. Click any invoice that has been sent (status: SENT) to open its detail page.
4. Scroll down to the **Transmission & Integration** panel at the bottom of the page.
5. You will see buttons including **Fetch External Attachments**.
6. Click **Fetch External Attachments** to request the list of attachments the external invoicing service has on file for this invoice.
7. The panel will display the attachments retrieved from the external service, including the filename, MIME type, and `attachmentIdentifier` code.

**Confirms requirement:** The user can fetch and view external attachments from the invoicing service for a given invoice, confirming that the integration with external services for attachment handling is working.

**UI Status:** ✅ VISIBLE

### AC8 — Batch-level attachments — attach one document to entire batch

1. Click **Invoices** in the top navigation bar.
2. In the left sidebar, click **Invoice Runs**.
3. Select **Real Run** mode.
4. The **Batch PDF Attachment (optional)** panel is clearly labelled: "Attach a single PDF document to every invoice in this batch via the external invoicing broker identifier."
5. Enter the identifier in **Attachment Identifier** and click **Start Real Run**.

**Confirms requirement:** The UI explicitly supports defining a single batch-level attachment that will be applied to every invoice in the run, eliminating the need to attach the file to each invoice individually.

**UI Status:** ✅ VISIBLE

### AC9 — AttachmentDetails in FINVOICE message ensuring correct linking in external system

Same as AC3 and AC5 — the FINVOICE XML download on the Invoice Detail page is the only UI surface for verifying this. The attachment identifier displayed in the Attachments list (`attachmentIdentifier` shown in small code font next to each attachment) also provides a reference.

1. Click **Invoices**, then **Invoice List** in the sidebar.
2. Click an invoice from the batch run.
3. Scroll to the **Attachments** section. Any attachment that has an `attachmentIdentifier` will show it as a small code label next to the filename.
4. Click **↓ FINVOICE XML** to download and verify the `AttachmentDetails` block in the FINVOICE message.

**Confirms requirement:** The identifier is visible in the attachments list and in the FINVOICE XML, but there is no parsed AttachmentDetails panel shown on-screen.

**UI Status:** ⚠️ PARTIAL

**What is missing:** No dedicated on-screen summary of the FINVOICE `AttachmentDetails` section on the Invoice Detail or Run Detail pages.

---

### AC10 — Automatic validation of attachment metadata before submission

1. Click **Invoices**, then **Invoice Runs** in the sidebar.
2. Select **Real Run** mode.
3. Enter an **Attachment Identifier** in the Batch PDF Attachment panel.
4. Click **Start Real Run**.
5. If the backend detects that the attachment identifier is invalid or the metadata is incompatible, an error message is displayed on the Invoice Run page in red below the form: for example, "Failed to start run" or a specific validation message from the server.

**Confirms requirement:** The UI will surface a server-side validation error if attachment metadata is rejected. However, there is no client-side pre-validation of the identifier format or MIME type before submission — the system only validates server-side after submission.

**UI Status:** ⚠️ PARTIAL

**What is missing:** No client-side validation of the attachment identifier or MIME type format before clicking **Start Real Run**. The MIME Type field has no dropdown or enforced format, and the identifier field has no format check. Errors are caught server-side only, after the full run submission.

---

### AC11 — Audit and compliance: logs all invoice–attachment associations

There is no UI surface for viewing an audit log of invoice–attachment associations. The Invoice Detail page shows a general **Reporting Data Audit Trail** table (for reporting field changes), but this does not include attachment association events.

**Confirms requirement:** This criterion has no visible UI. The backend logs attachment associations internally for compliance, but there is no screen or panel where a user can view those logs.

**UI Status:** ❌ NOT VISIBLE

**Why there is no UI:** The logging is handled server-side as part of the FINVOICE message generation and transmission process. No frontend component queries or displays this audit data.

---

### AC12 — Error handling: flags mismatched attachment identifier before submission

1. Click **Invoices**, then **Invoice Runs** in the sidebar.
2. Select **Real Run** mode and fill in an attachment identifier that does not exist in the invoicing service.
3. Click **Start Real Run**.
4. If the backend validates the identifier against the external service and the identifier is not found, an error message appears on the page: "Failed to start run" plus any server-side message.

**Confirms requirement:** The system will display an error when the run fails due to an invalid identifier. However, the error only appears after submission — there is no pre-submission check that validates the identifier against the external invoicing service before the user clicks **Start Real Run**.

**UI Status:** ⚠️ PARTIAL

**What is missing:** No pre-submission validation that checks whether the attachment identifier exists in the external invoicing service. The user cannot confirm the identifier is valid until the run has already been attempted.

---

## Backend-Only Aspects

**AC11 — Audit log of invoice–attachment associations:** The backend records all invoice–attachment linkages generated in each FINVOICE message for compliance and traceability. There is no user-facing screen for viewing this audit log. This is intentional because the log is an internal compliance record, not an operational workflow.
