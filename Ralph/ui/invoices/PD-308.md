## Where to Find This Feature

This feature spans two areas of the app. The language code is set on a customer's billing profile (Customers & Config tab), and the results of that setting — translated product names and the language tag on the invoice — are visible on the Invoice Detail page (Invoices tab) and managed in Master Data → Products.

## User Flows by Acceptance Criterion

---

### AC1 — System selects correct language version from customer profile

1. Click **Invoices** in the top navigation bar.
2. In the left sidebar, click **Invoice List**.
3. You will see a table of generated invoices. Click the row for any invoice you want to inspect.
4. On the invoice detail page, look at the header section at the top of the page. You will see a field labelled **Language** showing the language code (e.g. "fi", "sv", or "en") that was inherited from the customer's billing profile at the time the invoice was generated.
5. The product names shown in the invoice line items reflect the translations stored for that language.

**Confirms requirement:** The **Language** field in the invoice header confirms that the system recorded which language was applied when the invoice data was built.

**UI Status:** ✅ VISIBLE

---

### AC2 — Language integrated into external system; default is Finnish

1. Click **Customers & Config** in the top navigation bar.
2. In the left sidebar, click **Billing Profiles**.
3. You will see a table of customers. Click any customer row to open their billing profile.
4. On the billing profile page, scroll down to the **Billing Profile** section. Find the **Language Code** field. When a new customer is set up, this field defaults to **fi** (Finnish).
5. Click **Save Billing Profile** to persist the setting. The language code is then included in the invoice data sent to the external invoicing system.

**Confirms requirement:** The default value "fi" in the Language Code field and its inclusion in the saved payload confirms the system defaults to Finnish and passes the value to the external system.

**UI Status:** ✅ VISIBLE

---

### AC3 — Language code can be defined for a customer (Finnish, Swedish, or English)

1. Click **Customers & Config** in the top navigation bar.
2. In the left sidebar, click **Billing Profiles**.
3. Click any customer row to open their billing profile.
4. Scroll to the **Language Code** field under the Billing Profile section.
5. Type the desired language code into the text input: **fi** for Finnish, **sv** for Swedish, or **en** for English.
6. Click **Save Billing Profile**. A green success message confirms the setting was saved.

**Confirms requirement:** The Language Code field allows entry of fi, sv, or en values which are then stored and used on invoices.

**UI Status:** ⚠️ PARTIAL

The Language Code field is a plain free-text input rather than a dropdown constrained to the three valid options (fi, sv, en). A user could type any arbitrary value (e.g. "de" or "fr") and it would be accepted without validation. A dropdown listing Finnish (fi), Swedish (sv), and English (en) would better enforce the requirement's intent that only these three codes are permitted.

---

### AC4 — Products have separate names defined for each language

1. Click **Master Data** in the top navigation bar.
2. In the left sidebar, click **Products**.
3. You will see a table of all products. Each row has a **Translations** button showing the count of translations currently defined (e.g. "2 translations ▼").
4. Click that button on any product row to expand the translations panel below it.
5. The panel shows a table of existing translations with columns: **Locale** (e.g. FI, SV, EN) and **Name** (the translated product name for that locale).
6. To add or update a translation, use the **Add / Update Translation** form at the bottom of the panel: select a locale from the dropdown (Finnish (fi), Swedish (sv), or English (en)), enter the translated name, and click **Save**.
7. The translation list updates immediately. The locale filter at the top of the Products page lets you view only products that have a translation for a chosen language.

**Confirms requirement:** Each product can store a separate name per locale (fi, sv, en), and these are displayed and editable directly in the Products page.

**UI Status:** ✅ VISIBLE

---

### AC5 — Translations are defined for product units

There is no screen in the application where a user can define locale-specific labels for product pricing units (PCS, KG, TON, M3, LITER, METER, HOUR). The pricing unit is selected from a fixed code list (a dropdown in the Add/Edit Product modal) and is stored and displayed as a single universal code regardless of the customer's language.

**Confirms requirement:** Not confirmed — no UI surface for unit translations exists.

**UI Status:** ❌ NOT VISIBLE

There is no frontend screen for defining language-specific labels for product units. The unit values (PCS, KG, TON, etc.) are fixed codes. If the FINVOICE export maps these codes to localised text, that mapping is handled entirely in the backend with no user-facing configuration screen.

---

### AC6 — Addresses can be stored in two languages

1. Click **Customers & Config** in the top navigation bar.
2. In the left sidebar, click **Billing Profiles**.
3. Click any customer row to open their billing profile.
4. Scroll down to the **Billing Address** section.
5. You will see the primary address fields: **Street Address**, **Postal Code**, **City**, and **Country Code**. Enter the address in the primary language (Finnish by default).
6. Below those fields you will see the alternative address fields: **Alt Street Address**, **Alt City**, and **Alt Country Code**. Enter the address in the second language (e.g. Swedish) here.
7. Click **Save Billing Profile**. Both address versions are saved and the FINVOICE export uses the version that matches the customer's language setting.

**Confirms requirement:** The billing profile form stores two complete address versions, enabling the system to export the correct-language address in the FINVOICE file.

**UI Status:** ✅ VISIBLE

---

### AC7 — Invoice template language set per customer and exported in FINVOICE

1. Click **Invoices** in the top navigation bar.
2. In the left sidebar, click **Invoice List**.
3. Click any invoice row to open the invoice detail page.
4. In the header section at the top of the page, the **Language** field shows the language code that was applied to this invoice (derived from the customer's billing profile).
5. On the invoice detail page, scroll to the **Attachments** panel and click **↓ FINVOICE XML** to download the raw FINVOICE file. The file will contain the language tag matching the customer's language setting.

**Confirms requirement:** The Language field on the invoice header confirms the template language was set per customer. The downloadable FINVOICE XML carries that language setting to the external system.

**UI Status:** ✅ VISIBLE

## Backend-Only Aspects

**AC1 (partial backend)** — The actual selection of the translated product name during invoice generation (i.e., picking the right translation from the database for the customer's locale) happens in the backend service. The UI surfaces the result (the language tag on the invoice header and the translated names in the line items) but the selection logic is server-side.

**AC5 — Product unit translations** — If localised unit labels are mapped in the FINVOICE export, this mapping is defined in backend code or configuration with no user-facing screen in the frontend.
