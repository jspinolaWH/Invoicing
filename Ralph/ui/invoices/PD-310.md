## Where to Find This Feature

This feature spans two main areas of the app. The FINVOICE data is generated automatically when running an invoice run (under the Invoices tab), and the resulting XML can be downloaded from the Invoice Detail page. Supporting master data for VAT rates and accounting identifiers lives under the Master Data tab.

### AC1 — FINVOICE material is generated during an invoicing run

1. Click **Invoices** in the top navigation bar.
2. In the left sidebar, click **Invoice Runs**.
3. You will see the Invoice Run configuration page with a mode selector ("Real Run" or "Simulate First") and a filter form.
4. Select **Real Run** (the default) and fill in at least a billing period in the filter form.
5. Optionally fill in the "Batch PDF Attachment" section if you want a PDF attached to every invoice in the batch.
6. Click **Start Real Run**.
7. You will be redirected to the Invoice Run Detail page showing the run's progress. When processing completes, you will see a summary of how many invoices were generated.
8. The FINVOICE data is automatically generated for each invoice during this run — no additional step is needed.

**Confirms requirement:** The Invoice Run page starts the run and the system generates FINVOICE data for every invoice in the background, which can then be downloaded per invoice.

**UI Status:** ✅ VISIBLE

### AC4 — Dataset with mandatory FINVOICE fields; single invoice line may split across ledger accounts

1. Click **Invoices** in the top navigation bar.
2. In the left sidebar, click **Invoice List**.
3. You will see a table of all invoices. Click any row to open the Invoice Detail page.
4. On the Invoice Detail page, scroll down to the **Line Items** section. You will see a table with the following columns: Description, Qty, Unit Price, VAT %, Net, Gross, Classification, **Account**, and **Cost Center**.
5. Each line item shows the accounting account code and cost center code that will be embedded in the FINVOICE export. A customer-visible invoice line can appear as multiple rows here — one per accounting split — so that each sub-line carries its own ledger account identifier.
6. To confirm the FINVOICE XML contains these fields, click the **↓ FINVOICE XML** button in the **Attachments** section (also on the Invoice Detail page). The browser will download the XML file.

**Confirms requirement:** The line items table exposes the Account and Cost Center per line, showing how a single visible invoice line can be split across different ledger accounts. The downloadable FINVOICE XML contains these mandatory accounting fields.

**UI Status:** ✅ VISIBLE

### AC5 — FINVOICE compatibility: generate FINVOICE format and transmit to external system with accounting identifiers (cost center, service responsibility, waste type)

1. Click **Invoices** in the top navigation bar.
2. In the left sidebar, click **Invoice List**.
3. Click any invoice row to open the Invoice Detail page.
4. Scroll down to the **Attachments** section. You will see a **↓ FINVOICE XML** button. Click it to download the raw FINVOICE XML for this invoice — this is the file that gets transmitted to the external invoicing system.
5. Scroll further to the **Transmission & Integration** section.
6. If the invoice status is "Ready", click **Transmit to Operator** to send the FINVOICE data to the external invoicing broker. The panel will show the external reference number and transmission timestamp once complete.
7. If the invoice has already been sent, you can click **Check Delivery Status** to verify the external system received it.

**Confirms requirement:** The FINVOICE XML download button proves the format is generated. The Transmit to Operator button proves it is sent to the external invoicing system. The line items table shows accounting identifiers (account code and cost center code) embedded per line.

**UI Status:** ✅ VISIBLE

### AC6 — Supports multiple VAT rates with decimal precision; applies correct VAT rate

**Part A — Managing VAT rates:**

1. Click **Master Data** in the top navigation bar.
2. In the left sidebar, click **VAT Rates**.
3. You will see a table of all configured VAT rates showing Code, Rate (%), Valid From, Valid To, and who created each rate.
4. The Rate field supports decimal values (e.g. 24.00, 10.50, 0.00) — you can verify this by clicking **+ Add VAT Rate** and entering a rate with decimal places in the "Rate (%)" field (step size is 0.01).
5. Set a "Valid From" date so the system resolves the correct rate based on the billing event date.
6. Click **Save** to store the VAT rate. Multiple rates with different codes can coexist.

**Part B — Verifying correct VAT rate on an invoice:**

1. Click **Invoices** in the top navigation bar.
2. In the left sidebar, click **Invoice List** and click any invoice row.
3. On the Invoice Detail page, look at the **Line Items** table. The "VAT %" column shows the exact decimal VAT rate applied to each line item.
4. The **Invoice Amount Summary** at the bottom shows the total Net, VAT Amount, and Gross, confirming the system computed and applied the correct VAT.

**Confirms requirement:** The VAT Rates page lets administrators define multiple named VAT rates with decimal precision and date-based validity. The invoice line items table shows the applied VAT rate per line, confirming the correct rate was resolved by event date.

**UI Status:** ✅ VISIBLE

---

### AC7 — Integration with external systems

1. Click **Invoices** in the top navigation bar.
2. In the left sidebar, click **Invoice List** and click any invoice row.
3. Scroll to the **Transmission & Integration** panel on the Invoice Detail page. Depending on invoice status, you will see buttons: **Transmit to Operator**, **Check Delivery Status**, **View Invoice Image (PDF)**, **Fetch External Attachments**, and **Retransmit (Updated Address)**.
4. If the invoice is in "Ready" status, click **Transmit to Operator** to push the FINVOICE data to the external e-invoice broker. A confirmation panel shows the external reference and timestamp.
5. After transmission, click **Check Delivery Status** to query the external system for a delivery confirmation.

**For e-invoice operator management:**

6. Click **Tools** in the top navigation bar.
7. In the left sidebar, click **E-Invoice Op.** (E-Invoice Operator Management).
8. You will see the daily batch management page. The daily batch runs automatically at 02:00 to register new e-invoice customers (START) and deregister customers whose delivery method changed (TERMINATE).
9. Click **Trigger Batch Now** and confirm to run the batch immediately. The result panel shows how many registrations Started, Terminated, or Failed.

**Confirms requirement:** The Transmission & Integration panel on the Invoice Detail page and the E-Invoice Operator Management page together confirm the system is integrated with the external invoicing operator and supports both individual invoice transmission and batch registration management.

**UI Status:** ✅ VISIBLE
