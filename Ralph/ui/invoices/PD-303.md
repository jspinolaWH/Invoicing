## Where to Find This Feature

This feature lives under the **Invoices** tab. When a user opens any invoice from the Invoice List, an **Attachments** panel at the bottom of the invoice detail page displays all files sent with that invoice, including inline previews for images and PDFs.

---

## User Flows by Acceptance Criterion

---

### AC1 — Attachment retrieval API: attachments load automatically when invoice is opened

1. Click **Invoices** in the top navigation bar.
2. In the left sidebar, click **Invoice List**.
3. You will see a table of invoices. Click on any invoice row to open its detail page.
4. Scroll down to the bottom of the invoice detail page. You will see a section labelled **Attachments**.
5. The panel automatically fetches and lists all attachments associated with the invoice, displaying each file's name, size, and a download button.

**Confirms requirement:** The attachments panel loads file data from the API immediately upon opening the invoice, demonstrating that the retrieval mechanism works end-to-end.

**UI Status:** ✅ VISIBLE

---

### AC2 — Metadata linking: attachment details (filename, identifier, type, size) are displayed

1. Click **Invoices** in the top navigation bar.
2. In the left sidebar, click **Invoice List**.
3. Click on any invoice row.
4. Scroll to the **Attachments** section at the bottom.
5. Each attachment row shows: the **filename** in bold, the **file size** in kilobytes, the **security class** (if set), and the **attachment identifier** code (the FINVOICE `AttachmentDetails` identifier) displayed in a monospace style.

**Confirms requirement:** The metadata fields — filename, attachment identifier, file type (implicit from MIME type used for preview), and size — are all displayed in the attachment list row.

**UI Status:** ✅ VISIBLE

---

### AC3 — Secure storage and role-based access control

This criterion is enforced entirely in the backend. The API endpoint for retrieving attachments requires an authenticated session, and access control is applied server-side. There is no dedicated UI surface for configuring or displaying access control rules for attachments.

**Confirms requirement:** N/A — no user-facing screen is expected for this criterion; it is a backend integration requirement.

**UI Status:** ❌ NOT VISIBLE

[NOT VISIBLE — This is a backend-only security concern. The ERP backend controls who may call the attachment endpoints. No UI is needed or expected.]

---

### AC4 — Real-time on-demand retrieval when user selects an invoice for review

1. Click **Invoices** in the top navigation bar.
2. In the left sidebar, click **Invoice List**.
3. Click on any invoice row to open the detail view.
4. The **Attachments** panel loads immediately and dynamically fetches attachment data from the external system via the API — the attachments are not pre-cached; they are retrieved on demand each time the page is opened.
5. For PDF and image attachments, an inline preview renders automatically in the panel (a PDF viewer embedded as an iframe, or an image displayed directly).

**Confirms requirement:** The system retrieves attachments dynamically at the moment the user opens the invoice, satisfying the real-time on-demand retrieval requirement. Inline previews appear automatically for supported formats.

**UI Status:** ✅ VISIBLE

---

### AC5 — Standardised attachment formats: PDF/A, JPEG, and PNG supported with inline preview

1. Click **Invoices** in the top navigation bar.
2. In the left sidebar, click **Invoice List**.
3. Click on any invoice row.
4. Scroll to the **Attachments** section.
5. For any attachment that is a **PDF**: an embedded PDF viewer (iframe) appears below the attachment metadata row, showing the document inline at 400 px height.
6. For any attachment that is a **JPEG** or **PNG** image: the image renders inline at full available width below the attachment row.
7. The upload zone (below the list) also enforces format restrictions — it only accepts `.pdf`, `.jpg`, `.jpeg`, and `.png` files, and will show an error message if an unsupported file type is chosen.

**Confirms requirement:** All three standardised formats (PDF/A, JPEG, PNG) render as inline previews in the Attachments panel, and only those formats are accepted for upload.

**UI Status:** ✅ VISIBLE

---

### AC6 — Error handling: users see an appropriate message when an attachment is missing or cannot be retrieved

1. Click **Invoices** in the top navigation bar.
2. In the left sidebar, click **Invoice List**.
3. Click on any invoice row.
4. Scroll to the **Attachments** section.
5. If an invoice has no attachments, the panel displays the text **"No attachments."** in a muted style.
6. If a file download fails (e.g., the user clicks **Download** and the file is unavailable), a browser alert popup says **"Download failed."**
7. If the **FINVOICE XML** button is clicked and the XML is not available, a browser alert says **"FINVOICE XML not available for this invoice."**

**Confirms requirement:** Basic error messaging exists. However, the error handling uses raw browser `alert()` popups rather than structured in-page error messages, and there is no distinct status indicator per attachment (e.g., "unavailable" badge) if a specific attachment fails to load its preview silently.

**UI Status:** ⚠️ PARTIAL

[PARTIAL — Download errors and XML errors show a browser `alert()` popup, which is functional but not a structured in-page error message as the requirement describes. Silent failures (preview fetch errors) are swallowed with no user-visible feedback. A proper in-page error banner or per-attachment status indicator is missing.]

---

### Core Requirement — Attachments sent with the invoice are visible when the invoice is opened

1. Click **Invoices** in the top navigation bar.
2. In the left sidebar, click **Invoice List**.
3. You will see a table listing all invoices. Click any invoice row.
4. The invoice detail page opens, showing the invoice number, header information, line items, amounts, and custom text.
5. Scroll down past the custom text section to find the **Attachments** panel.
6. All attachments sent with the invoice are listed. PDF attachments show an embedded document viewer; image attachments show an inline image. Each entry shows the filename, size in KB, security class, and attachment identifier.
7. Click **Download** on any attachment to save it locally.
8. Click **↓ FINVOICE XML** in the top-right of the Attachments panel to download the raw FINVOICE XML file for the invoice.

**Confirms requirement:** The invoicing specialist can open a sent invoice and immediately see and review all attachments exactly as sent, satisfying the core review requirement.

**UI Status:** ✅ VISIBLE
