### Gap: AC19 — No admin UI for configuring or extending the visible set of filter criteria

**AC:** Additional configurable criteria; visible selection fields defined
**Layer:** Frontend
**UI Surface:** Invoices → Invoice Runs (RunFilterForm)
**What is missing:** The Invoice Runs filter form has a fixed, hardcoded set of fields (Municipality, Min Amount, Period From/To, Service Responsibility, Location, Customer Type, Service / Product Code, Billing Frequency, Invoice Template, Invoice Number Series). There is no administrator screen for dynamically adding new custom filter fields, toggling which criteria are visible in the filter form, or configuring which fields appear in the invoice printing view. The requirement states that "other fields can also be configured as criteria" and "fields to be visible need to be defined" — but no such configuration UI exists. The available criteria are determined by the code, not by any user-controlled configuration.
