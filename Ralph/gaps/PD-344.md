## Frontend UI Gaps

---

### Gap 1 — AC2: No dedicated filter for shared-service or waste-reception origin type on the Billing Events list

**Layer:** Frontend

**AC:** AC2 — Events related to shared services and waste reception must also be transferable

**What is missing:** The Billing Events list page does not provide a dedicated filter for the "Shared Service" or "Waste Reception" origin types. Users who need to find and bulk-transfer all shared-service events for a customer must manually scan the "Origin" column or apply the Service Responsibility text filter as an approximation. There is no Origin dropdown filter in the filter bar that would let a user directly select "SHARED_SERVICE" or "DRIVER" (waste reception) to produce a clean event set for bulk transfer. This makes the shared-service transfer use case harder to execute correctly than the general transfer use case, despite the backend supporting it fully.
