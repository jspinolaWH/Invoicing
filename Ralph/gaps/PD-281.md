## UI Gaps

### Gap: AC2 — Sync result does not confirm how many open invoices were updated

**Layer:** Frontend
**AC:** AC2 — Updated billing address applied to all open invoices and payment reminders
**What is missing:** After a successful address sync, the Address Sync result panel shows a status of UPDATED confirming the customer record was changed, but it does not tell the user how many (or which) open invoices received the updated billing address. Users must manually inspect individual invoice detail pages to verify the address change propagated to open invoices. There is no count, list, or summary of "N invoices updated" in the sync response.
**UI surface needed:** The Address Sync result panel should display a count of open invoices whose billing address was updated (e.g. "3 open invoices updated") and optionally list them by invoice number, so the user has confirmation that the address propagated correctly without manual verification.

---

### Gap: AC4 — No manager-change history or before/after comparison in the UI

**Layer:** Frontend
**AC:** AC4 — Property manager change: reminders and open invoices go to new manager
**What is missing:** The UI provides no dedicated view showing a manager-change history or confirming that in-flight payment reminders targeting the old manager were redirected to the new manager. Users must manually verify by inspecting reminder recipients on each invoice individually. There is no filter or report showing "reminders sent before vs. after manager change" nor any before/after address comparison on the sync result page.
**UI surface needed:** A "Recent Address Changes" section on the Address Sync page or the Billing Profile Change History tab that shows the previous and new address values with a timestamp, and ideally a count of pending reminders that were redirected as a result of the change.

---

### Gap: AC7 — No invoice-level confirmation that new billing details were applied to open invoices

**Layer:** Frontend
**AC:** AC7 — New billing details are applied to open invoices
**What is missing:** After a sync or manual billing profile save, the backend propagates the new address to open invoices (by clearing finvoiceXml), but the UI does not show any explicit "N invoices updated" confirmation in the sync result or billing profile save response. Users have no way to know how many open invoices were affected by the update without opening each one individually.
**UI surface needed:** The billing profile save success message and the Address Sync result panel should both include a count of open invoices updated (e.g. "Billing profile saved. 5 open invoices updated with new address."), giving users clear confirmation that the change propagated.

---

### Gap: AC11 — Sync result does not distinguish local update from broker-level update

**Layer:** Frontend
**AC:** AC11 — Update billing info via interface on open invoices already transferred to invoice broker
**What is missing:** The sync result panel shows UPDATED or SKIPPED_LOCKED, but does not distinguish between "updated on our side only" and "updated at the invoice broker too." There is no transmit-update or re-push button to explicitly resend the corrected address to the broker for individual invoices that have already been transmitted. Users cannot confirm from the UI whether already-transmitted invoices will reflect the new address when the broker next processes them.
**UI surface needed:** The sync result panel should explicitly state whether already-transmitted (SENT) invoices were included in the update and flag them. A "Re-transmit with updated address" action on individual SENT invoice detail pages would give users a direct mechanism to push the corrected address to the broker for specific invoices.
