## UI Gaps

### Gap: AC3 — No UI visibility into automatic e-invoice address retrieval from integration

**Layer:** Frontend
**AC:** AC3 — E-invoicing addresses are also retrieved automatically via integration (no user approval required)
**What is missing:** The automatic retrieval of e-invoice addresses via the operator integration is a backend-only operation with no frontend surface. Users have no way to see when the integration last ran, which addresses were updated, or whether an address was populated by the integration vs. entered manually. The only visible evidence is that the address field may be pre-populated when the user next opens the Billing Profile page — but there is no timestamp, source indicator, or activity log for integration-driven updates.
**UI surface needed:** A "Last updated by integration" timestamp or source label next to the E-Invoice Address field on the Billing Profile page, and/or a log entry on the Change History tab indicating when the integration last wrote or attempted to write an e-invoice address for the customer. This allows users to distinguish manually entered addresses from integration-sourced ones and to audit integration activity.
