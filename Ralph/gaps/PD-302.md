## Gaps

### Gap 1 — AC6: Custom text changes not tracked in the visible audit trail

**Layer:** Frontend

**AC:** AC6 — Audit and accounting: text stored as part of invoicing history

**What is missing from the UI:** The existing Reporting Data Audit Trail on the Invoice Detail page does not show custom text changes (who set or changed the text, what the previous value was, when it changed). The custom text is persisted on the Invoice entity and covered by `BaseAuditEntity` timestamps, but changes to the `customText` field are not surfaced in a visible change log in the UI. A user reviewing the invoice history has no way to see that the custom text was modified, by whom, or from what prior value.

**UI surface description:** Invoice Detail page → Reporting Data Audit Trail section (bottom of page). The audit table columns are: Line Item, Field, Assigned Value, Logged At, Logged By. This table tracks line-item reporting field changes only. No entry is ever shown for custom text saves (clicking "Save Text" in `InvoiceCustomTextPanel.jsx`).

### Gap 2 — AC7: Invoice template editor has no custom text field or preview

**Layer:** Frontend

**AC:** AC7 — Seamless template integration: custom text integrates with invoice templates

**What is missing from the UI:** The Invoice Templates page (Master Data → Invoice Templates) exists and allows creating and editing templates (Name, Code, Number Series), but the template editor does not expose a custom text field or preview showing how the custom text integrates with the rendered template output. Users cannot see how the custom text will be positioned or rendered within a specific template from the UI. The integration is handled at the backend level (customText is embedded in the FINVOICE XML via the `InvoiceFreeText` element), but there is no UI step where a user can preview or configure the text placement within a template.

**UI surface description:** Master Data → Invoice Templates page (`InvoiceTemplatePage.jsx`). The template create/edit form has fields for Name, Code, and Number Series only. There is no "Custom Text placeholder" field, no template preview panel, and no indication of where the custom text will appear in the rendered output.
