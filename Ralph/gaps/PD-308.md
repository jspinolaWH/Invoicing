# Gap Report: PD-308

**Requirement:** 3.4.4 Invoice data based on language selection
**Scanned:** 2026-04-21T00:00:00Z
**Verdict:** PASSES

## Acceptance Criteria Coverage

| # | Criterion | Status | Notes |
|---|-----------|--------|-------|
| 1 | When generating invoice data, the system selects the correct language version based on the language setting saved in the customer's profile. | IMPLEMENTED | `InvoiceGenerationService.resolveLanguage()` reads `customer.getBillingProfile().getLanguageCode()` and maps it to `InvoiceLanguage` enum; set on generated invoice. |
| 2 | The selected language is integrated into the external invoicing system in the same way as other criteria and elements defined for the invoice template. The system's default primary language is Finnish. | IMPLEMENTED | `FinvoiceBuilderService` exports `DocumentLanguage` element with invoice language. `resolveLanguage()` defaults to `InvoiceLanguage.FI`; `BillingProfileValidationService` warns if language code is missing. |
| 3 | A language code can be defined for a customer account (Finnish, Swedish, or English). | IMPLEMENTED | `BillingProfile.languageCode` column (fi/sv/en). `BillingProfileRequest` accepts it via API. Frontend `BillingProfilePage` includes a language code input defaulting to 'fi'. |
| 4 | Products have separate names defined for each language. | IMPLEMENTED | `ProductTranslation` entity with `locale` + `name` fields; unique constraint on (product_id, locale). `ProductService.addOrUpdateTranslation()` manages them. `ProductsPage.jsx` provides full CRUD translation UI for fi/sv/en. |
| 5 | Translations are defined for product units. | IMPLEMENTED | `PricingUnit` enum has `getLabel(String language)` returning locale-specific unit labels (e.g. PCS → kpl/st/pcs). `FinvoiceBuilderService.resolveUnitCode()` uses invoice language to select the correct label. |
| 6 | Addresses can be stored in two languages, and the address is exported to the FINVOICE file according to the selected language. | IMPLEMENTED | `BillingAddress` has `streetAddressAlt`, `cityAlt`, `countryCodeAlt` fields. `FinvoiceBuilderService.appendBuyerPartyDetails()` uses alt fields when invoice language ≠ FI and alt fields are populated. |
| 7 | The system sets the invoice template language according to the customer's selection, and exports this in the FINVOICE file. | IMPLEMENTED | `FinvoiceBuilderService.appendInvoiceDetails()` adds `<DocumentLanguage>` element. If `invoice.getTemplateCode()` is set it is also exported as `InvoiceTypeCode IdentifierType="TemplateCode"`. |
