## Gaps Found

### Gap 1: UI does not communicate the credit note numbering rule to the user
- **Layer**: Frontend
- **Criterion**: AC 4 — credit note is assigned the number of the invoice being credited
- **What exists**: The backend correctly sets the credit note's invoice number to match the original invoice number (`creditNote.setInvoiceNumber(original.getInvoiceNumber())`). The credit note detail page header displays the credit note's number, and the original invoice page shows a "partially/fully credited" banner with a link to the credit note.
- **What is missing**: The UI displays the credit note number but does not explain to the user that the number is derived from the credited invoice's own number — this business rule is invisible to the user. A user relying only on the UI cannot verify or trust the numbering without manually cross-referencing. The credit note detail page should include an explicit label or tooltip (e.g. "Credit note number is assigned from the original invoice number per numbering policy") so that the rule is transparent and auditable without backend access.
- **Files to change**:
  - `invoicing-fe/src/pages/invoices/InvoiceDetailPage.jsx` — on the credit note variant of the page, add a tooltip or explanatory note near the invoice number field stating that the credit note carries the same number as the credited invoice (e.g. a small info icon with "Number matches credited invoice per policy")
  - `invoicing-fe/src/pages/invoices/CreditNoteForm.jsx` — optionally, after the credit note is issued and the user is redirected, surface the assigned number with a brief explanation in the success state
