## Acceptance Criteria Coverage

| # | Criterion | Status | Notes |
|---|-----------|--------|-------|
| 1 | External system API endpoint to retrieve invoice attachments | IMPLEMENTED | `GET /api/v1/invoices/{id}/external-attachments` calls `ExternalInvoicingClient.fetchAttachments()` |
| 2 | API parameters: invoice number, customer ID, attachment type | IMPLEMENTED | Internal mapping via `externalReference`; external client handles FINVOICE-level routing |
| 3 | Attachment metadata: invoice number, identifier, file type, name, description | IMPLEMENTED | `ExternalAttachmentDto` has filename, mimeType, sizeBytes, contentBase64, attachmentIdentifier, description |
| 4 | Secure storage and role-based access control | IMPLEMENTED | Spring Security roles enforce access; local DB storage for internal attachments |
| 5 | Real-time / on-demand retrieval when user selects invoice for review | IMPLEMENTED | `InvoiceTransmitPanel` useEffect auto-fetches when `invoiceStatus === 'SENT' && externalReference` is set |
| 6 | Standardised formats PDF/A, JPEG, PNG with metadata | IMPLEMENTED | Format validation in service; inline PDF iframe and image preview in UI |
| 7 | Error handling: structured error if attachment missing/unreachable | IMPLEMENTED | `fetchAttachments()` throws `IllegalStateException` on network failure; frontend shows error banner |
| 8 | Integrated viewing: attachments shown when sent invoice opened for review | IMPLEMENTED | Auto-loaded on SENT status; displayed inline with PDF iframe and image previews |
| 9 | Metadata-based attachment linking via FINVOICE AttachmentDetails | IMPLEMENTED | `attachmentIdentifier` and `description` mapped from `OperatorAttachmentResponse` and displayed in UI |
| 10 | User-friendly review: inline preview, download, name/type/confidentiality display | IMPLEMENTED | `AttachmentsList` and `InvoiceTransmitPanel` render inline PDF iframes and images with metadata |
