## Gaps

### Gap 1 — AC9: No explicit pre-transmission attachment validation gate visible to the user

**Layer:** Frontend

**AC:** AC9 — ERP ensures all attachments are validated before sending externally (FINVOICE)

**What is missing from the UI:** The upload-time validation is clear and visible. However, there is no explicit "validate attachments" step shown to the user as a distinct pre-transmission gate — validation happens automatically on upload. The FINVOICE XML button provides indirect confirmation but does not surface a pass/fail compliance summary to the user. A user cannot tell at a glance whether all current attachments are FINVOICE-compliant before hitting transmit.

**UI surface description:** Invoice Detail page → Attachments panel (`InvoiceAttachmentsPanel.jsx`). The panel shows uploaded files and inline errors on upload, but has no compliance status indicator or pre-transmit validation summary panel between the Attachments section and the Transmit section.

### Gap 2 — AC13: Base64 encoding and SHA1 generation steps are invisible to the user in the upload flow

**Layer:** Frontend

**AC:** AC13 — Use case: upload two PDFs, Base64 encode, generate SHA1, validate, transmit

**What is missing from the UI:** The upload, file count, size tracking, error messaging, and security classification steps are all visible. The Base64 encoding and SHA1 generation are backend-only (not shown as explicit steps in the UI). While this is architecturally intentional, the UI provides no confirmation feedback (e.g., a "processing…" indicator during encoding or a checksum-confirmed badge after upload) that these mandatory FINVOICE compliance steps completed successfully. A user has no way to confirm the attachment was properly encoded without downloading and inspecting the FINVOICE XML manually.

**UI surface description:** Invoice Detail page → Attachments panel → attachment list row (`AttachmentsList.jsx`). Each row shows filename, size, and security class but does not display a SHA1 checksum confirmation badge or encoding-complete indicator after a successful upload.
