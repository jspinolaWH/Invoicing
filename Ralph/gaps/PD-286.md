## Gaps Found

### Gap 1 — AC1: No per-customer exemption report or mid-season visibility in the UI
**AC:** AC1 — Contract-date compliance: prevent overbilling for mid-season contracts
**Layer:** Frontend
**What is missing:** The Min. Fees configuration page exposes the two contract-date exemption checkboxes (Exempt if contract started after period start / Exempt if contract ended before period end), but there is no per-customer override screen and no UI report showing which specific customers were exempted due to these rules during a run. After an invoice run completes, users have no way to see a list of customers who were shielded from the minimum fee because of mid-season contract dates. The logic runs correctly in the backend, but the result is entirely invisible in the frontend.

---

### Gap 2 — AC2: Automatic minimum fee calculation has no UI visibility
**AC:** AC2 — Automatic minimum fee calculation: add surcharge if invoice falls below minimum
**Layer:** Frontend
**What is missing:** There is no dedicated screen, progress indicator, log, or user-facing step that shows the minimum fee comparison calculation taking place. The automatic comparison of the invoice net total against the configured threshold and the injection of a top-up line item happens entirely in the backend invoicing engine. The only evidence visible to the user is the resulting adjustment line item on the completed invoice (AC6). Users cannot see which invoices triggered a minimum fee top-up, or the before/after net amounts, in any run summary or simulation results screen.

---

### Gap 3 — AC4: No per-customer contract validation report in the UI
**AC:** AC4 — Customer contract validation before applying minimum charge
**Layer:** Frontend
**What is missing:** The Min. Fees configuration page allows administrators to set the contract-date validation flags (system-level), but there is no customer-level screen where a user can see whether a specific customer passed or failed the contract validation, and there is no run-level report listing which customers were exempted due to contract dates. After a run, users cannot determine from the UI which customers had their minimum fee waived because their contract started or ended mid-period — this information is computed and applied by the backend but never exposed to the frontend.
