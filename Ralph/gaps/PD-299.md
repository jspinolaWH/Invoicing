## Frontend Gaps (added from UI synthesis)

---

### Gap: AC4 — No UI surface for FINVOICE export configuration or inspection

**AC:** Event data included in FINVOICE material per FINVOICE specification
**Layer:** Frontend
**UI Surface:** Billing Event Detail page (Operations → Billing Events → View)
**What is missing:** There is no user-facing screen where the FINVOICE output of an event can be inspected or configured. The FINVOICE formatting happens entirely in the backend when an event transitions to the transmission stage. Users can observe that events are transmitted (status changes to SENT) but there is no UI that exposes the raw FINVOICE XML, shows which billing event fields map to which FINVOICE elements, or confirms which event data was included in the generated material.

---

### Gap: AC5 — Multi-VAT / multi-cost-centre combination not surfaced at event level

**AC:** Same invoice can include multiple VAT rates and cost centres
**Layer:** Frontend
**UI Surface:** Billing Event Detail page → Details tab → VAT Details panel
**What is missing:** The billing event detail page shows only the single resolved VAT rate and cost centre for that individual event. The fact that an invoice can aggregate multiple events with different VAT rates and cost centres is only visible at the invoice level (Invoice Detail → Line Items). The billing event detail page itself does not show how the event's rate and cost centre combine with sibling events on the same invoice, nor does it link to the parent invoice to allow cross-referencing. A tester can only confirm multi-rate support by navigating away to the invoice.

---

### Gap: AC6 — No UI for weighbridge/POS event creation or integration monitoring

**AC:** Event data can be generated via weighbridge or cashier/POS integration
**Layer:** Frontend
**UI Surface:** Operations → Billing Events
**What is missing:** There is no UI screen where a user can trigger a weighbridge import, configure POS/cashier event ingestion, or simulate an integration-sourced event creation. Integration-sourced events appear in the Billing Events list with their Origin badge once they arrive, but the creation and ingestion side of the integration has no user-facing screen. The WeighbridgeConfigPage exists at `/integration/weighbridge-config` but is not linked in the main sidebar navigation, making it unreachable through normal app navigation.
