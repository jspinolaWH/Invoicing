## UI Gaps

### Gap: AC5 — Enforcement mechanism routing not surfaced in the UI

**Layer:** Frontend
**AC:** AC5 — Ensures correct legal collection processes, VAT application, and enforcement mechanisms under the appropriate legal framework
**What is missing:** The billing event detail page shows the classification and VAT details but does not display which enforcement authority or collection path was selected for the invoice based on its classification. Enforcement mechanism routing is applied automatically by the backend (via FINVOICE classification routing), but the user has no way to see or verify which enforcement path was chosen for a specific invoice. There is no enforcement routing panel, field, or read-only indicator anywhere in the UI.
**UI surface needed:** A read-only "Enforcement Path" or "Collection Authority" indicator on the Billing Event Detail page and/or Invoice Detail page showing which enforcement channel (e.g. statutory authority for PUBLIC_LAW, general debt collection for PRIVATE_LAW) applies to this record.
