## Frontend Gaps (added from UI synthesis)

---

### Gap: AC3 — No UI restriction or guidance for cost center format changes requiring support

**AC:** Structure and content of cost centers defined during implementation; WasteHero support assists with format changes
**Layer:** Frontend
**UI Surface:** Master Data → Cost Centers
**What is missing:** The Cost Centers page allows main users to directly edit all three segment fields (product segment, reception segment, responsibility segment) and structurally change the composite code format without any restriction or guidance. The AC implies that structural format changes require WasteHero support assistance, but the UI shows no "contact support" prompt, no field-level lock for format-defining properties, and no informational note explaining that segment structure changes should be coordinated with support. Users can change the cost center format structure unilaterally with no guardrail.

---

### Gap: AC13 — No UI for cash register system integration

**AC:** Cash register system (system designation — data source)
**Layer:** Frontend
**UI Surface:** No screen exists
**What is missing:** There is no frontend page for configuring, monitoring, or interacting with a cash register system integration. Billing data originating from cash registers is ingested by the backend via an integration pipeline with no user-facing screen. Users cannot see the connection status, configure the cash register endpoint, view ingestion logs, or trigger a manual sync. The integration is set up at the infrastructure level during implementation with no UI representation.

---

### Gap: AC14 — No UI for weighing system integration

**AC:** Weighing system (system designation — data source)
**Layer:** Frontend
**UI Surface:** No screen exists in main navigation (WeighbridgeConfigPage exists at `/integration/weighbridge-config` but is not linked in the sidebar)
**What is missing:** There is no user-facing page in the main application navigation for weighing system integration. The `/integration/weighbridge-config` route exists in App.jsx but is not linked from any sidebar section, making it invisible to users navigating the app normally. Users cannot reach weighbridge configuration through the standard UI flow. Additionally, there is no page for monitoring weighbridge data ingestion, viewing connection status, or triggering manual syncs.
