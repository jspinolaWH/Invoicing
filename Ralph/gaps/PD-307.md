## Gaps Found

### Gap 1: InvoiceRunResponse missing templateId/numberSeriesId; InvoiceRunDetail missing template display

- **Layer**: Both
- **Criterion**: AC5 (template selection during invoicing run), AC6 (template code sent to external system — no UI confirmation)
- **What exists**: `InvoiceRun.templateId` and `InvoiceRun.numberSeriesId` stored on the entity; `RunFilterForm.jsx` sends `templateId` when creating a run; backend resolves and embeds template code in FINVOICE output.
- **What is missing**:
  - Backend: `InvoiceRunResponse` does not expose `templateId` or `numberSeriesId` fields; `from()` mapper does not map them.
  - Frontend: `InvoiceRunDetailPage` does not show which template/number series was configured for the run. Users cannot see what template was applied (AC5) or confirm it was sent to the external system (AC6 partial).
- **Files to change**:
  - `invoicing/src/main/java/com/example/invoicing/entity/invoicerun/dto/InvoiceRunResponse.java` — add `private Long templateId;` and `private Long numberSeriesId;` fields, update `from()` mapper
  - `invoicing-fe/src/pages/runs/InvoiceRunDetailPage.jsx` — display the template name/code and number series on the run detail view
