## Gaps Found

### Gap 1 — AC3: No dedicated property-as-project creation flow in the frontend
**AC:** AC3 — Projects as separate properties linked to the customer
**Layer:** Frontend
**What is missing:** The frontend does not offer a dedicated flow for creating a new property and explicitly tagging it as a project. The billing event detail page shows the resolved Project field for property-linked project IDs, but there is no standalone UI page or form where a user can create a property and designate it as a project. Properties currently arrive via integration data or backend seeding — the frontend exposes the data (read-only on the event detail page) but provides no surface for a user to create or manage the property-to-project association directly. This makes the "separate properties" creation pathway invisible and inaccessible to office users through the application UI.
