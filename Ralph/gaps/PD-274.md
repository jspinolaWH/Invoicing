## Gaps Found

### Gap 1: No simulation-phase audit trail surfaced in the UI
- **Layer**: Frontend
- **Criterion**: AC 6 (criterion 9) — audit and compliance support during simulation
- **What exists**: The backend correctly guards audit log writes behind the `simulationMode` flag so no spurious entries are created. Invoice-level audit trails are visible on the Invoice Detail page for committed real runs. The Simulation Results page displays validation outcomes (BLOCKING / WARNING errors by category and per customer).
- **What is missing**: The Simulation Results page and the Invoice Run Detail page do not expose a timestamped, step-by-step log of what was checked during the simulation phase — what order checks ran, when, and with what outcomes. The requirement calls for full traceability during simulation; the current UI surfaces validation outcomes only after the fact, not a traceable audit trail of the simulation process itself.
- **Files to change**:
  - `invoicing-fe/src/pages/runs/SimulationResultsPage.jsx` — add a collapsible "Simulation Audit Log" section that calls a new API endpoint returning timestamped simulation-phase events
  - `invoicing-fe/src/pages/runs/InvoiceRunDetailPage.jsx` — surface simulation-phase audit entries alongside post-commit entries when the run originated as a simulation
  - `invoicing/src/main/java/com/example/invoicing/service/InvoiceSimulationService.java` — emit structured audit events for each simulation step (event retrieval, pricing, validation) into a simulation-scoped log store
  - `invoicing/src/main/java/com/example/invoicing/controller/run/SimulationController.java` — add GET endpoint returning simulation audit log for a given run ID
